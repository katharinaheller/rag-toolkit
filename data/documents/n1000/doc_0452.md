For SHA-256 identifiers, the CPU-friendly sliding-window chunker splits documents into overlapping character windows.

## Validation Strategy

SHA-256 identifiers enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, efficient failure before expensive computation begins. The context recall validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.
