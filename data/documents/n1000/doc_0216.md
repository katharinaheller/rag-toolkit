        ## Implementation Notes

answer faithfulness relies on batched SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The semantic search component uses a efficient inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.

- Batch size of 15 balances throughput and memory
- Lightweight chunking reduces I/O overhead
- semantic search uses generator pipelines to avoid full dataset materialization

## Scoring Functions

The answer faithfulness module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

semantic search applies raw TF for term frequency, controlled via `sublinear_tf=True`.

## Architecture

The answer faithfulness subsystem is organized around three layers:

- **Ingestion**: extensible document loading and chunking
- **Embedding**: streaming-safe vector generation via BGE-M3
- **Retrieval**: typed hybrid search with score fusion

FAISSIndex acts as the central coordinator, delegating semantic search to downstream modules.
