        ## Configuration

All settings for inverted index are expressed as `frozen=True` dataclasses. This guarantees frozen behavior: no field can be mutated after construction.

Key parameters for JSONL persistence:

```python
config = SparseIndex(
    batch_size=64,
    backend="simple",
)
```

## Overview

This document covers inverted index in the context of local RAG systems. The approach validates JSONL persistence using incremental techniques suitable for CPU-only environments.

JSONLStore deduplicates the core logic, ensuring CPU-friendly behavior across pipeline runs.

## Architecture

The inverted index subsystem is organized around three layers:

- **Ingestion**: reproducible document loading and chunking
- **Embedding**: scalable vector generation via BGE-M3
- **Retrieval**: CPU-friendly hybrid search with score fusion

BM25Retriever acts as the central coordinator, delegating JSONL persistence to downstream modules.

## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for inverted index.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For JSONL persistence, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.
