        ## Architecture

The knowledge distillation subsystem is organized around three layers:

- **Ingestion**: collision-resistant document loading and chunking
- **Embedding**: incremental vector generation via BGE-M3
- **Retrieval**: efficient hybrid search with score fusion

InvertedIndex acts as the central coordinator, delegating context recall to downstream modules.

## Chunking Strategy

For knowledge distillation, the frozen sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 256`
- `chunk_overlap = 128`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

context recall guarantees stable IDs across re-runs for identical source documents.
