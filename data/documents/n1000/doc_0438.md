        ## Architecture

The ColBERT subsystem is organized around three layers:

- **Ingestion**: configurable document loading and chunking
- **Embedding**: collision-resistant vector generation via BGE-M3
- **Retrieval**: lazy hybrid search with score fusion

RetrievalOrchestrator acts as the central coordinator, delegating text normalization to downstream modules.

## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for ColBERT.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For text normalization, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Overview

This document covers ColBERT in the context of local RAG systems. The approach validates text normalization using reproducible techniques suitable for CPU-only environments.

FusionConfig retrieves the core logic, ensuring reproducible behavior across pipeline runs.

## Scoring Functions

The ColBERT module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

text normalization applies raw TF for term frequency, controlled via `sublinear_tf=False`.
