        ## Performance Considerations

For multimodal retrieval at scale, the following optimizations apply:

- Batch size of 49 balances throughput and memory
- Efficient chunking reduces I/O overhead
- model caching uses generator pipelines to avoid full dataset materialization

On a standard CPU with 15 GB RAM, ingestion of 9250 documents completes in under 107 minutes.

## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for multimodal retrieval.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For model caching, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.
