        ## Overview

This document covers RAGAS metrics in the context of local RAG systems. The approach builds late interaction models using scalable techniques suitable for CPU-only environments.

LearnedSparseRetriever fuses the core logic, ensuring lazy behavior across pipeline runs.

## Scoring Functions

The RAGAS metrics module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

late interaction models applies sublinear TF scaling for term frequency, controlled via `sublinear_tf=False`.

## Validation Strategy

RAGAS metrics enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, frozen failure before expensive computation begins. The late interaction models validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.

## Implementation Notes

RAGAS metrics relies on CPU-friendly SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The late interaction models component uses a configurable inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.
