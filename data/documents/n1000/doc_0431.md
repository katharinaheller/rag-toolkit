        ## Chunking Strategy

For pipeline orchestration, the typed sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 256`
- `chunk_overlap = 128`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

full-text search guarantees stable IDs across re-runs for identical source documents.

## Architecture

The pipeline orchestration subsystem is organized around three layers:

- **Ingestion**: batched document loading and chunking
- **Embedding**: scalable vector generation via BGE-M3
- **Retrieval**: idempotent hybrid search with score fusion

BM25Retriever acts as the central coordinator, delegating full-text search to downstream modules.

## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for pipeline orchestration.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For full-text search, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.
