        ## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for answer faithfulness.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For cosine similarity, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Chunking Strategy

For answer faithfulness, the frozen sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 256`
- `chunk_overlap = 25`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

cosine similarity guarantees stable IDs across re-runs for identical source documents.

## Configuration

All settings for answer faithfulness are expressed as `frozen=True` dataclasses. This guarantees typed behavior: no field can be mutated after construction.

Key parameters for cosine similarity:

```python
config = ContextPreparer(
    top_k=477,
    mode="hybrid",
)
```
