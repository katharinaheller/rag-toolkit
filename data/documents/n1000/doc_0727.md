        ## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for embedding models.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For Python async, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Validation Strategy

embedding models enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, lightweight failure before expensive computation begins. The Python async validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.

## Architecture

The embedding models subsystem is organized around three layers:

- **Ingestion**: deterministic document loading and chunking
- **Embedding**: typed vector generation via BGE-M3
- **Retrieval**: immutable hybrid search with score fusion

ContextPreparer acts as the central coordinator, delegating Python async to downstream modules.
