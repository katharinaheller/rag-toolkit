        ## Implementation Notes

bfloat16 precision relies on modular SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The MRL embeddings component uses a fail-fast inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.

## Architecture

The bfloat16 precision subsystem is organized around three layers:

- **Ingestion**: scalable document loading and chunking
- **Embedding**: modular vector generation via BGE-M3
- **Retrieval**: scalable hybrid search with score fusion

ChunkStore acts as the central coordinator, delegating MRL embeddings to downstream modules.

## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for bfloat16 precision.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For MRL embeddings, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.
