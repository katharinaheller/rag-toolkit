        ## Architecture

The thread safety subsystem is organized around three layers:

## Retrieval Pipeline

The thread safety retrieval pipeline operates in five stages:

1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-169 from each leg
3. **Score fusion**: weighted sum with normalize_scores=True
4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: lexical + length baseline

document store ensures no text or metadata enters the index backends directly.
