        ## Architecture

The chain-of-thought prompting subsystem is organized around three layers:

- **Ingestion**: defensive document loading and chunking
- **Embedding**: validated vector generation via BGE-M3
- **Retrieval**: efficient hybrid search with score fusion

OllamaClient acts as the central coordinator, delegating query expansion to downstream modules.

## Performance Considerations

- Batch size of 22 balances throughput and memory
- Modular chunking reduces I/O overhead
- query expansion uses generator pipelines to avoid full dataset materialization

## Chunking Strategy

For chain-of-thought prompting, the streaming-safe sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 500`
- `chunk_overlap = 25`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

query expansion guarantees stable IDs across re-runs for identical source documents.
