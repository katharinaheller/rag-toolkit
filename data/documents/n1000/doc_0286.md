        ## Chunking Strategy

For data preprocessing, the modular sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 500`
- `chunk_overlap = 64`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

few-shot learning guarantees stable IDs across re-runs for identical source documents.

## Configuration

All settings for data preprocessing are expressed as `frozen=True` dataclasses. This guarantees efficient behavior: no field can be mutated after construction.

Key parameters for few-shot learning:

```python
config = EmbeddingStore(
    batch_size=98,
    backend="hybrid",
)
```

## Validation Strategy

data preprocessing enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, deterministic failure before expensive computation begins. The few-shot learning validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.
