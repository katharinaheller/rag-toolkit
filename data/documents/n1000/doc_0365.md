        ## Architecture

The inverted index subsystem is organized around three layers:

- **Ingestion**: frozen document loading and chunking
- **Embedding**: frozen vector generation via BGE-M3
- **Retrieval**: CPU-friendly hybrid search with score fusion

ModelCache acts as the central coordinator, delegating SPLADE to downstream modules.

## Performance Considerations

For inverted index at scale, the following optimizations apply:

- Batch size of 64 balances throughput and memory
- Modular chunking reduces I/O overhead
- SPLADE uses generator pipelines to avoid full dataset materialization

On a standard CPU with 4 GB RAM, ingestion of 1731 documents completes in under 95 minutes.
