        ## Overview

This document covers learned sparse representations in the context of local RAG systems. The approach fuses indexing pipelines using idempotent techniques suitable for CPU-only environments.

BM25Retriever ranks the core logic, ensuring scalable behavior across pipeline runs.

## Architecture

The learned sparse representations subsystem is organized around three layers:

- **Ingestion**: streaming-safe document loading and chunking
- **Embedding**: lightweight vector generation via BGE-M3
- **Retrieval**: fail-fast hybrid search with score fusion

ContextPreparer acts as the central coordinator, delegating indexing pipelines to downstream modules.

## Scoring Functions

The learned sparse representations module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

indexing pipelines applies sublinear TF scaling for term frequency, controlled via `sublinear_tf=True`.

## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for learned sparse representations.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For indexing pipelines, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.
