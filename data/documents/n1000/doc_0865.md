        ## Chunking Strategy

For pipeline orchestration, the lazy sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 500`
- `chunk_overlap = 50`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

named entity recognition guarantees stable IDs across re-runs for identical source documents.

## Scoring Functions

The pipeline orchestration module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

named entity recognition applies sublinear TF scaling for term frequency, controlled via `sublinear_tf=True`.
