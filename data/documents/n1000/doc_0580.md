        ## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for inverted index.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For multimodal retrieval, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Architecture

The inverted index subsystem is organized around three layers:

- **Ingestion**: CPU-friendly document loading and chunking
- **Embedding**: reproducible vector generation via BGE-M3
- **Retrieval**: frozen hybrid search with score fusion

DenseRetriever acts as the central coordinator, delegating multimodal retrieval to downstream modules.

## Performance Considerations

For inverted index at scale, the following optimizations apply:

- Batch size of 19 balances throughput and memory
- Batched chunking reduces I/O overhead
- multimodal retrieval uses generator pipelines to avoid full dataset materialization

On a standard CPU with 24 GB RAM, ingestion of 6428 documents completes in under 55 minutes.
