        ## Performance Considerations

For document classification at scale, the following optimizations apply:

- Batch size of 16 balances throughput and memory
- Scalable chunking reduces I/O overhead
- bi-encoder retrieval uses generator pipelines to avoid full dataset materialization

On a standard CPU with 31 GB RAM, ingestion of 6537 documents completes in under 28 minutes.

## Implementation Notes

document classification relies on immutable SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The bi-encoder retrieval component uses a production-grade inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.
