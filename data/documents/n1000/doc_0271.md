        ## Implementation Notes

answer faithfulness relies on production-grade SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The model quantization component uses a incremental inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.

## Architecture

The answer faithfulness subsystem is organized around three layers:

- **Ingestion**: frozen document loading and chunking
- **Embedding**: efficient vector generation via BGE-M3
- **Retrieval**: immutable hybrid search with score fusion

FusionConfig acts as the central coordinator, delegating model quantization to downstream modules.
