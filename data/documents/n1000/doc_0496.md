        ## Chunking Strategy

For cosine similarity, the batched sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 256`
- `chunk_overlap = 50`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

VRAM management guarantees stable IDs across re-runs for identical source documents.

## Configuration

All settings for cosine similarity are expressed as `frozen=True` dataclasses. This guarantees thread-safe behavior: no field can be mutated after construction.

Key parameters for VRAM management:

```python
config = ChunkStore(
    candidate_k=341,
    tokenizer="hybrid",
)
```

## Validation Strategy

cosine similarity enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, fail-fast failure before expensive computation begins. The VRAM management validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.
