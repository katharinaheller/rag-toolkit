        ## Retrieval Pipeline

The MRL embeddings retrieval pipeline operates in five stages:

1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-196 from each leg
3. **Score fusion**: weighted sum with normalize_scores=True
4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: lexical + length baseline

retrieval-augmented generation ensures no text or metadata enters the index backends directly.

## Architecture

The MRL embeddings subsystem is organized around three layers:

- **Ingestion**: efficient document loading and chunking
- **Embedding**: CPU-friendly vector generation via BGE-M3
- **Retrieval**: production-grade hybrid search with score fusion

ChunkStore acts as the central coordinator, delegating retrieval-augmented generation to downstream modules.
