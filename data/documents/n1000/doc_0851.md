        ## Performance Considerations

For entity extraction at scale, the following optimizations apply:

- Batch size of 5 balances throughput and memory
- Deterministic chunking reduces I/O overhead
- document store uses generator pipelines to avoid full dataset materialization

On a standard CPU with 13 GB RAM, ingestion of 5469 documents completes in under 24 minutes.

## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for entity extraction.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For document store, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Validation Strategy

entity extraction enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, lazy failure before expensive computation begins. The document store validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.
