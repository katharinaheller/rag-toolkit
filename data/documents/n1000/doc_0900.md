        ## Overview

This document covers open-source models in the context of local RAG systems. The approach builds full-text search using scalable techniques suitable for CPU-only environments.

DocumentStore serializes the core logic, ensuring fail-fast behavior across pipeline runs.

## Retrieval Pipeline

The open-source models retrieval pipeline operates in five stages:

1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-161 from each leg
3. **Score fusion**: RRF with normalize_scores=True
4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: disabled

full-text search ensures no text or metadata enters the index backends directly.

## Validation Strategy

open-source models enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, configurable failure before expensive computation begins. The full-text search validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.
