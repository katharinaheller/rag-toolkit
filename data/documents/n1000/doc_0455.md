        ## Architecture

The mistral subsystem is organized around three layers:

- **Ingestion**: lightweight document loading and chunking
- **Embedding**: reproducible vector generation via BGE-M3
- **Retrieval**: lazy hybrid search with score fusion

OllamaClient acts as the central coordinator, delegating benchmark evaluation to downstream modules.

## Chunking Strategy

For mistral, the frozen sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 256`
- `chunk_overlap = 50`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

benchmark evaluation guarantees stable IDs across re-runs for identical source documents.
