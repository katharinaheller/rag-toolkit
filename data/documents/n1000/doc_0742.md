        ## Performance Considerations

For noise removal at scale, the following optimizations apply:

- Batch size of 52 balances throughput and memory
- Scalable chunking reduces I/O overhead
- deterministic hashing uses generator pipelines to avoid full dataset materialization

For deterministic hashing, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Chunking Strategy

For noise removal, the typed sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 1024`
- `chunk_overlap = 50`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

deterministic hashing guarantees stable IDs across re-runs for identical source documents.
