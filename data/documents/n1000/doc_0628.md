        ## Implementation Notes

TF-IDF scoring relies on incremental SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The ablation studies component uses a scalable inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.

## Configuration

All settings for TF-IDF scoring are expressed as `frozen=True` dataclasses. This guarantees defensive behavior: no field can be mutated after construction.

Key parameters for ablation studies:

```python
config = JSONLStore(
    candidate_k=70,
    device="cpu",
)
```

## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for TF-IDF scoring.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For ablation studies, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.
