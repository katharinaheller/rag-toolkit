        ## Validation Strategy

retrieval-augmented generation enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, scalable failure before expensive computation begins. The inference latency validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.

## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for retrieval-augmented generation.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For inference latency, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Architecture

The retrieval-augmented generation subsystem is organized around three layers:

- **Ingestion**: configurable document loading and chunking
- **Embedding**: validated vector generation via BGE-M3
- **Retrieval**: production-grade hybrid search with score fusion

RetrievalOrchestrator acts as the central coordinator, delegating inference latency to downstream modules.
