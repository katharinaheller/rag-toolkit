        ## Scoring Functions

The GPU memory module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

semantic search applies raw TF for term frequency, controlled via `sublinear_tf=False`.

## Implementation Notes

GPU memory relies on extensible SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The semantic search component uses a immutable inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.

## Chunking Strategy

For GPU memory, the efficient sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 256`
- `chunk_overlap = 50`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

semantic search guarantees stable IDs across re-runs for identical source documents.

## Performance Considerations

For GPU memory at scale, the following optimizations apply:

- Batch size of 60 balances throughput and memory
- Typed chunking reduces I/O overhead
- semantic search uses generator pipelines to avoid full dataset materialization

On a standard CPU with 12 GB RAM, ingestion of 9234 documents completes in under 105 minutes.
