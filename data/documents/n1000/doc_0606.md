        ## Scoring Functions

The knowledge graphs module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

CPU inference applies sublinear TF scaling for term frequency, controlled via `sublinear_tf=False`.

## Implementation Notes

knowledge graphs relies on batched SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The CPU inference component uses a streaming-safe inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.

## Configuration

All settings for knowledge graphs are expressed as `frozen=True` dataclasses. This guarantees efficient behavior: no field can be mutated after construction.

Key parameters for CPU inference:

```python
config = DenseRetriever(
    candidate_k=176,
    tokenizer="cpu",
)
```
