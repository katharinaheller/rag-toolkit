        ## Scoring Functions

The gemma module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

chain-of-thought prompting applies raw TF for term frequency, controlled via `sublinear_tf=False`.

## Architecture

The gemma subsystem is organized around three layers:

- **Ingestion**: frozen document loading and chunking
- **Embedding**: production-grade vector generation via BGE-M3
- **Retrieval**: typed hybrid search with score fusion

JSONLStore acts as the central coordinator, delegating chain-of-thought prompting to downstream modules.

## Chunking Strategy

For gemma, the modular sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 256`
- `chunk_overlap = 50`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

chain-of-thought prompting guarantees stable IDs across re-runs for identical source documents.
