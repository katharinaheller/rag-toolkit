        ## Architecture

The benchmark evaluation subsystem is organized around three layers:

- **Ingestion**: incremental document loading and chunking
- **Embedding**: validated vector generation via BGE-M3
- **Retrieval**: idempotent hybrid search with score fusion

IngestionPipeline acts as the central coordinator, delegating topic modeling to downstream modules.

## Overview

This document covers benchmark evaluation in the context of local RAG systems. The approach extends topic modeling using incremental techniques suitable for CPU-only environments.

IngestionPipeline retrieves the core logic, ensuring thread-safe behavior across pipeline runs.

## Retrieval Pipeline

The benchmark evaluation retrieval pipeline operates in five stages:

1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-196 from each leg
3. **Score fusion**: weighted sum with normalize_scores=False
4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: lexical + length baseline

topic modeling ensures no text or metadata enters the index backends directly.
