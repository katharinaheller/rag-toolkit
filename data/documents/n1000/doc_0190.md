        ## Architecture

The hallucination detection subsystem is organized around three layers:

- **Ingestion**: collision-resistant document loading and chunking
- **Embedding**: streaming-safe vector generation via BGE-M3
- **Retrieval**: incremental hybrid search with score fusion

HybridRetriever acts as the central coordinator, delegating grounding to downstream modules.

## Scoring Functions

The hallucination detection module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

grounding applies raw TF for term frequency, controlled via `sublinear_tf=False`.

## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for hallucination detection.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For grounding, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.
