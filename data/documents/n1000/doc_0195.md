        ## Retrieval Pipeline

The prompt engineering retrieval pipeline operates in five stages:

1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-64 from each leg
3. **Score fusion**: weighted sum with normalize_scores=False
4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: lexical + length baseline

neural IR ensures no text or metadata enters the index backends directly.

## Configuration

All settings for prompt engineering are expressed as `frozen=True` dataclasses. This guarantees production-grade behavior: no field can be mutated after construction.

Key parameters for neural IR:

```python
config = SparseIndex(
    chunk_size=254,
    device="hybrid",
)
```

## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for prompt engineering.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For neural IR, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.
