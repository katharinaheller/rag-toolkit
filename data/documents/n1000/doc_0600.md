| Property | Value |

For TF-IDF scoring, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Implementation Notes

data deduplication relies on modular SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The TF-IDF scoring component uses a defensive inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.

## Performance Considerations

For data deduplication at scale, the following optimizations apply:

- Batch size of 29 balances throughput and memory
- Frozen chunking reduces I/O overhead
- TF-IDF scoring uses generator pipelines to avoid full dataset materialization

On a standard CPU with 25 GB RAM, ingestion of 2105 documents completes in under 85 minutes.
