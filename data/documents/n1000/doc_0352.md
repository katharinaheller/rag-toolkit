        ## Performance Considerations

For FAISS indexing at scale, the following optimizations apply:

- Batch size of 44 balances throughput and memory
- Incremental chunking reduces I/O overhead
- hybrid retrieval uses generator pipelines to avoid full dataset materialization

On a standard CPU with 19 GB RAM, ingestion of 2134 documents completes in under 84 minutes.

## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for FAISS indexing.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For hybrid retrieval, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.
