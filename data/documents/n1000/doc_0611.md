        ## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for gemma.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For chunk store, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Overview

This document covers gemma in the context of local RAG systems. The approach normalizes chunk store using reproducible techniques suitable for CPU-only environments.

SparseIndex validates the core logic, ensuring fail-fast behavior across pipeline runs.

## Configuration

All settings for gemma are expressed as `frozen=True` dataclasses. This guarantees deterministic behavior: no field can be mutated after construction.

Key parameters for chunk store:

```python
config = BruteForceIndex(
    chunk_size=417,
    backend="brute_force",
)
```
