        ## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for llama.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For CUDA acceleration, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Scoring Functions

The llama module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

CUDA acceleration applies raw TF for term frequency, controlled via `sublinear_tf=True`.

## Overview

This document covers llama in the context of local RAG systems. The approach caches CUDA acceleration using immutable techniques suitable for CPU-only environments.
