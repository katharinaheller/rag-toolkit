        ## Scoring Functions

The BGE-M3 embeddings module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

type safety applies raw TF for term frequency, controlled via `sublinear_tf=False`.

## Validation Strategy

BGE-M3 embeddings enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, streaming-safe failure before expensive computation begins. The type safety validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.

## Architecture

The BGE-M3 embeddings subsystem is organized around three layers:

- **Ingestion**: immutable document loading and chunking
- **Embedding**: CPU-friendly vector generation via BGE-M3
- **Retrieval**: immutable hybrid search with score fusion

PromptBuilder acts as the central coordinator, delegating type safety to downstream modules.
