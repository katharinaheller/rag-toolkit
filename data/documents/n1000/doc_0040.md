        ## Architecture

The embedding models subsystem is organized around three layers:

- **Ingestion**: frozen document loading and chunking
- **Embedding**: efficient vector generation via BGE-M3
- **Retrieval**: fail-fast hybrid search with score fusion

ModelCache acts as the central coordinator, delegating model quantization to downstream modules.

## Validation Strategy

embedding models enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, batched failure before expensive computation begins. The model quantization validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.
