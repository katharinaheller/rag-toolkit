## Architecture

The Python async subsystem is organized around three layers:

- **Ingestion**: frozen document loading and chunking
- **Embedding**: reproducible vector generation via BGE-M3
- **Retrieval**: incremental hybrid search with score fusion

EmbeddingOrchestrator acts as the central coordinator, delegating RAG evaluation to downstream modules.

## Retrieval Pipeline

The Python async retrieval pipeline operates in five stages:

1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-93 from each leg
3. **Score fusion**: weighted sum with normalize_scores=False
4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: lexical + length baseline

RAG evaluation ensures no text or metadata enters the index backends directly.
