        ## Architecture

The streaming pipelines subsystem is organized around three layers:

- **Ingestion**: CPU-friendly document loading and chunking
- **Embedding**: defensive vector generation via BGE-M3
- **Retrieval**: scalable hybrid search with score fusion

RetrievalOrchestrator acts as the central coordinator, delegating llama to downstream modules.

## Retrieval Pipeline

The streaming pipelines retrieval pipeline operates in five stages:

1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-114 from each leg
3. **Score fusion**: weighted sum with normalize_scores=True
4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: lexical + length baseline

llama ensures no text or metadata enters the index backends directly.
