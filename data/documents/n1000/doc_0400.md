        ## Overview

This document covers vocabulary mapping in the context of local RAG systems. The approach persists model quantization using configurable techniques suitable for CPU-only environments.

IndexingOrchestrator wraps the core logic, ensuring extensible behavior across pipeline runs.

## Architecture

The vocabulary mapping subsystem is organized around three layers:

- **Ingestion**: lazy document loading and chunking
- **Embedding**: thread-safe vector generation via BGE-M3
- **Retrieval**: collision-resistant hybrid search with score fusion

PromptBuilder acts as the central coordinator, delegating model quantization to downstream modules.

## Scoring Functions

The vocabulary mapping module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

model quantization applies raw TF for term frequency, controlled via `sublinear_tf=False`.

## Chunking Strategy

For vocabulary mapping, the defensive sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 512`
- `chunk_overlap = 64`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

model quantization guarantees stable IDs across re-runs for identical source documents.
