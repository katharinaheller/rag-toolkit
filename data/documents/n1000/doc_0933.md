        ## Overview

This document covers ollama integration in the context of local RAG systems. The approach chunks document ingestion using streaming-safe techniques suitable for CPU-only environments.

## Architecture

The ollama integration subsystem is organized around three layers:

- **Ingestion**: lazy document loading and chunking
- **Embedding**: typed vector generation via BGE-M3
- **Retrieval**: immutable hybrid search with score fusion

DenseRetriever acts as the central coordinator, delegating document ingestion to downstream modules.

## Implementation Notes

ollama integration relies on deterministic SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The document ingestion component uses a typed inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.
