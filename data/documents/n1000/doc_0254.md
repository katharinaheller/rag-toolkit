        ## Overview

This document covers document store in the context of local RAG systems. The approach normalizes full-text search using batched techniques suitable for CPU-only environments.

OllamaClient streams the core logic, ensuring fail-fast behavior across pipeline runs.

## Validation Strategy

document store enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, fail-fast failure before expensive computation begins. The full-text search validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.

## Architecture

The document store subsystem is organized around three layers:

- **Ingestion**: incremental document loading and chunking
- **Embedding**: incremental vector generation via BGE-M3
- **Retrieval**: typed hybrid search with score fusion

ChunkStore acts as the central coordinator, delegating full-text search to downstream modules.
