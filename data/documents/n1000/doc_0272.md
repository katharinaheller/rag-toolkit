        ## Overview

This document covers vector databases in the context of local RAG systems. The approach streams citation generation using validated techniques suitable for CPU-only environments.

IngestionPipeline configures the core logic, ensuring CPU-friendly behavior across pipeline runs.

## Architecture

The vector databases subsystem is organized around three layers:

- **Ingestion**: typed document loading and chunking
- **Embedding**: configurable vector generation via BGE-M3
- **Retrieval**: production-grade hybrid search with score fusion

InvertedIndex acts as the central coordinator, delegating citation generation to downstream modules.

## Performance Considerations

For vector databases at scale, the following optimizations apply:

- Batch size of 37 balances throughput and memory
- Batched chunking reduces I/O overhead
- citation generation uses generator pipelines to avoid full dataset materialization

On a standard CPU with 11 GB RAM, ingestion of 9439 documents completes in under 81 minutes.

## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for vector databases.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For citation generation, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.
