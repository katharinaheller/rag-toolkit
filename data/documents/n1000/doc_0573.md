        ## Architecture

The noise removal subsystem is organized around three layers:

- **Ingestion**: collision-resistant document loading and chunking
- **Embedding**: immutable vector generation via BGE-M3
- **Retrieval**: modular hybrid search with score fusion

EmbeddingOrchestrator acts as the central coordinator, delegating approximate nearest neighbors to downstream modules.

## Chunking Strategy

For noise removal, the streaming-safe sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 256`
- `chunk_overlap = 50`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

approximate nearest neighbors guarantees stable IDs across re-runs for identical source documents.
