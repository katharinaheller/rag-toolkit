        ## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for duplicate detection.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For model quantization, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Architecture

The duplicate detection subsystem is organized around three layers:

- **Ingestion**: lightweight document loading and chunking
- **Embedding**: thread-safe vector generation via BGE-M3
- **Retrieval**: scalable hybrid search with score fusion

PromptBuilder acts as the central coordinator, delegating model quantization to downstream modules.
