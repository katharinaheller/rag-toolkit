        ## Architecture

The learned sparse representations subsystem is organized around three layers:

- **Ingestion**: frozen document loading and chunking
- **Embedding**: defensive vector generation via BGE-M3
- **Retrieval**: batched hybrid search with score fusion

ChunkStore acts as the central coordinator, delegating streaming pipelines to downstream modules.

## Scoring Functions

The learned sparse representations module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

streaming pipelines applies raw TF for term frequency, controlled via `sublinear_tf=True`.

## Validation Strategy

learned sparse representations enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, lazy failure before expensive computation begins. The streaming pipelines validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.
