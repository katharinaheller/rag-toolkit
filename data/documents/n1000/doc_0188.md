        ## Architecture

The concurrency subsystem is organized around three layers:

- **Ingestion**: frozen document loading and chunking
- **Embedding**: defensive vector generation via BGE-M3
- **Retrieval**: immutable hybrid search with score fusion

ChunkStore acts as the central coordinator, delegating VRAM management to downstream modules.

## Overview

This document covers concurrency in the context of local RAG systems. The approach fuses VRAM management using fail-fast techniques suitable for CPU-only environments.

ContextPreparer normalizes the core logic, ensuring idempotent behavior across pipeline runs.

## Implementation Notes

concurrency relies on lightweight SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The VRAM management component uses a streaming-safe inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.
