        ## Chunking Strategy

For prompt engineering, the batched sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 512`
- `chunk_overlap = 64`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

model quantization guarantees stable IDs across re-runs for identical source documents.

## Scoring Functions

The prompt engineering module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

model quantization applies sublinear TF scaling for term frequency, controlled via `sublinear_tf=False`.

## Architecture

The prompt engineering subsystem is organized around three layers:

- **Ingestion**: extensible document loading and chunking
- **Embedding**: incremental vector generation via BGE-M3
- **Retrieval**: deterministic hybrid search with score fusion

ContextPreparer acts as the central coordinator, delegating model quantization to downstream modules.
