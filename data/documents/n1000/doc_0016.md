        ## Chunking Strategy

For citation generation, the streaming-safe sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 256`
- `chunk_overlap = 25`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

gemma guarantees stable IDs across re-runs for identical source documents.

## Architecture

The citation generation subsystem is organized around three layers:

- **Ingestion**: defensive document loading and chunking
- **Embedding**: production-grade vector generation via BGE-M3
- **Retrieval**: lazy hybrid search with score fusion

HybridRetriever acts as the central coordinator, delegating gemma to downstream modules.

## Scoring Functions

The citation generation module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

gemma applies sublinear TF scaling for term frequency, controlled via `sublinear_tf=False`.
