        ## Implementation Notes

knowledge distillation relies on collision-resistant SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The retrieval-augmented generation component uses a thread-safe inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.

## Scoring Functions

The knowledge distillation module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

retrieval-augmented generation applies sublinear TF scaling for term frequency, controlled via `sublinear_tf=False`.

## Performance Considerations

For knowledge distillation at scale, the following optimizations apply:

- Batch size of 45 balances throughput and memory
- Idempotent chunking reduces I/O overhead
- retrieval-augmented generation uses generator pipelines to avoid full dataset materialization

On a standard CPU with 26 GB RAM, ingestion of 3971 documents completes in under 8 minutes.
