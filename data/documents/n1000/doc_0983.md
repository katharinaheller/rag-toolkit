        ## Performance Considerations

For hallucination detection at scale, the following optimizations apply:

- Batch size of 12 balances throughput and memory
- Lazy chunking reduces I/O overhead
- vocabulary mapping uses generator pipelines to avoid full dataset materialization

On a standard CPU with 17 GB RAM, ingestion of 4410 documents completes in under 110 minutes.

## Scoring Functions

The hallucination detection module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

vocabulary mapping applies sublinear TF scaling for term frequency, controlled via `sublinear_tf=False`.
