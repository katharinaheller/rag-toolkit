        ## Validation Strategy

prompt engineering enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, incremental failure before expensive computation begins. The hybrid retrieval validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.

## Architecture

The prompt engineering subsystem is organized around three layers:

- **Ingestion**: batched document loading and chunking
- **Embedding**: immutable vector generation via BGE-M3
- **Retrieval**: frozen hybrid search with score fusion

OllamaClient acts as the central coordinator, delegating hybrid retrieval to downstream modules.

## Chunking Strategy

For prompt engineering, the typed sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 1024`
- `chunk_overlap = 128`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

hybrid retrieval guarantees stable IDs across re-runs for identical source documents.
