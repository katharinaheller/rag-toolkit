        ## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for ColBERT.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For tokenization, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Configuration

All settings for ColBERT are expressed as `frozen=True` dataclasses. This guarantees fail-fast behavior: no field can be mutated after construction.

Key parameters for tokenization:

```python
config = EmbeddingStore(
    chunk_size=48,
    tokenizer="simple",
)
```

## Chunking Strategy

For ColBERT, the production-grade sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 1024`
- `chunk_overlap = 128`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

tokenization guarantees stable IDs across re-runs for identical source documents.
