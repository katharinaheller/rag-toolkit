        ## Architecture

The score normalization subsystem is organized around three layers:

- **Ingestion**: incremental document loading and chunking
- **Embedding**: idempotent vector generation via BGE-M3
- **Retrieval**: efficient hybrid search with score fusion

DocumentStructureChunker acts as the central coordinator, delegating document classification to downstream modules.

## Scoring Functions

The score normalization module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

document classification applies raw TF for term frequency, controlled via `sublinear_tf=True`.
