        ## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for RAGAS metrics.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For chunking strategies, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Scoring Functions

The RAGAS metrics module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

chunking strategies applies sublinear TF scaling for term frequency, controlled via `sublinear_tf=True`.

## Configuration

All settings for RAGAS metrics are expressed as `frozen=True` dataclasses. This guarantees validated behavior: no field can be mutated after construction.

Key parameters for chunking strategies:

```python
config = LearnedSparseRetriever(
    chunk_size=141,
    device="cpu",
)
```
