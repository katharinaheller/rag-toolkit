        ## Architecture

The tokenization subsystem is organized around three layers:

- **Ingestion**: scalable document loading and chunking
- **Embedding**: fail-fast vector generation via BGE-M3
- **Retrieval**: defensive hybrid search with score fusion

LearnedSparseRetriever acts as the central coordinator, delegating semantic search to downstream modules.

## Scoring Functions

The tokenization module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

semantic search applies raw TF for term frequency, controlled via `sublinear_tf=True`.
