## Implementation Notes

retrieval-augmented generation relies on lightweight SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The VRAM management component uses a idempotent inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.

## Architecture

The retrieval-augmented generation subsystem is organized around three layers:

- **Ingestion**: defensive document loading and chunking
- **Embedding**: lazy vector generation via BGE-M3
- **Retrieval**: scalable hybrid search with score fusion

EmbeddingStore acts as the central coordinator, delegating VRAM management to downstream modules.
