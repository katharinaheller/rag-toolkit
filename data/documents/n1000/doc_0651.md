        ## Performance Considerations

For batch processing at scale, the following optimizations apply:

- Batch size of 9 balances throughput and memory
- Frozen chunking reduces I/O overhead
- few-shot learning uses generator pipelines to avoid full dataset materialization

On a standard CPU with 15 GB RAM, ingestion of 9581 documents completes in under 47 minutes.

## Scoring Functions

The batch processing module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

few-shot learning applies raw TF for term frequency, controlled via `sublinear_tf=False`.

## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for batch processing.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For few-shot learning, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.
