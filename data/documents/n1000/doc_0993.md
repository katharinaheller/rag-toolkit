        ## Validation Strategy

noise removal enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, reproducible failure before expensive computation begins. The knowledge graphs validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.

## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for noise removal.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For knowledge graphs, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Architecture

The noise removal subsystem is organized around three layers:

- **Ingestion**: deterministic document loading and chunking
- **Embedding**: validated vector generation via BGE-M3
- **Retrieval**: incremental hybrid search with score fusion

OllamaClient acts as the central coordinator, delegating knowledge graphs to downstream modules.
