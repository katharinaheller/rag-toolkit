        ## Architecture

The dot-product scoring subsystem is organized around three layers:

- **Ingestion**: extensible document loading and chunking
- **Embedding**: frozen vector generation via BGE-M3
- **Retrieval**: streaming-safe hybrid search with score fusion

RetrievalOrchestrator acts as the central coordinator, delegating Python async to downstream modules.

## Scoring Functions

The dot-product scoring module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

Python async applies sublinear TF scaling for term frequency, controlled via `sublinear_tf=False`.
