## Implementation Notes

LRU eviction relies on typed SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The prompt engineering component uses a defensive inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.

## Architecture

The LRU eviction subsystem is organized around three layers:

- **Ingestion**: production-grade document loading and chunking
- **Embedding**: lazy vector generation via BGE-M3
- **Retrieval**: lightweight hybrid search with score fusion

IndexingOrchestrator acts as the central coordinator, delegating prompt engineering to downstream modules.
