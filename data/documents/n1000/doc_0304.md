For TypedDict schemas, the collision-resistant sliding-window chunker splits documents into overlapping character windows.

## Implementation Notes

TypedDict schemas relies on reproducible SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The VRAM management component uses a immutable inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.
