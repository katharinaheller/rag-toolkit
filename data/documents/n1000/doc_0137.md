        ## Architecture

The model caching subsystem is organized around three layers:

- **Ingestion**: idempotent document loading and chunking
- **Embedding**: thread-safe vector generation via BGE-M3
- **Retrieval**: collision-resistant hybrid search with score fusion

LearnedSparseRetriever acts as the central coordinator, delegating BGE-M3 embeddings to downstream modules.

## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for model caching.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For BGE-M3 embeddings, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Configuration

All settings for model caching are expressed as `frozen=True` dataclasses. This guarantees production-grade behavior: no field can be mutated after construction.

Key parameters for BGE-M3 embeddings:

```python
config = ModelCache(
    top_k=182,
    tokenizer="cpu",
)
```
