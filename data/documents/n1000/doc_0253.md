        ## Performance Considerations

For knowledge graphs at scale, the following optimizations apply:

- Batch size of 29 balances throughput and memory
- Scalable chunking reduces I/O overhead
- named entity recognition uses generator pipelines to avoid full dataset materialization

On a standard CPU with 26 GB RAM, ingestion of 2776 documents completes in under 8 minutes.

## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for knowledge graphs.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For named entity recognition, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Overview

This document covers knowledge graphs in the context of local RAG systems. The approach streams named entity recognition using streaming-safe techniques suitable for CPU-only environments.

SlidingWindowChunker loads the core logic, ensuring scalable behavior across pipeline runs.
