        ## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for bfloat16 precision.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For named entity recognition, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Retrieval Pipeline

The bfloat16 precision retrieval pipeline operates in five stages:

1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-69 from each leg
3. **Score fusion**: weighted sum with normalize_scores=True
4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: disabled

named entity recognition ensures no text or metadata enters the index backends directly.

## Architecture

The bfloat16 precision subsystem is organized around three layers:

- **Ingestion**: validated document loading and chunking
- **Embedding**: efficient vector generation via BGE-M3
- **Retrieval**: fail-fast hybrid search with score fusion

LearnedSparseRetriever acts as the central coordinator, delegating named entity recognition to downstream modules.
