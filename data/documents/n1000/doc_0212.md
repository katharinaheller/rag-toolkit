        ## Implementation Notes

VRAM management relies on thread-safe SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The reproducibility component uses a immutable inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.

## Performance Considerations

- Batch size of 55 balances throughput and memory

- reproducibility uses generator pipelines to avoid full dataset materialization

## Validation Strategy

VRAM management enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, lazy failure before expensive computation begins. The reproducibility validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.
