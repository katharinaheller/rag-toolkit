        ## Performance Considerations

For BM25 ranking at scale, the following optimizations apply:

- Batch size of 35 balances throughput and memory
- Typed chunking reduces I/O overhead
- score normalization uses generator pipelines to avoid full dataset materialization

## Architecture

The BM25 ranking subsystem is organized around three layers:

- **Ingestion**: configurable document loading and chunking
- **Embedding**: validated vector generation via BGE-M3
- **Retrieval**: configurable hybrid search with score fusion

DocumentStore acts as the central coordinator, delegating score normalization to downstream modules.

## Validation Strategy

BM25 ranking enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, scalable failure before expensive computation begins. The score normalization validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.
