        ## Scoring Functions

The SPLADE module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

topic modeling applies raw TF for term frequency, controlled via `sublinear_tf=True`.

## Implementation Notes

SPLADE relies on thread-safe SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The topic modeling component uses a thread-safe inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.

## Chunking Strategy

For SPLADE, the lazy sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 500`
- `chunk_overlap = 128`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

topic modeling guarantees stable IDs across re-runs for identical source documents.

## Configuration

All settings for SPLADE are expressed as `frozen=True` dataclasses. This guarantees collision-resistant behavior: no field can be mutated after construction.
