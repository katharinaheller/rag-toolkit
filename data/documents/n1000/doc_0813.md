        ## Validation Strategy

zero-shot inference enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, lazy failure before expensive computation begins. The instruction tuning validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.

## Performance Considerations

For zero-shot inference at scale, the following optimizations apply:

- Batch size of 22 balances throughput and memory
- Cpu-friendly chunking reduces I/O overhead
- instruction tuning uses generator pipelines to avoid full dataset materialization

On a standard CPU with 16 GB RAM, ingestion of 9937 documents completes in under 113 minutes.

## Implementation Notes

zero-shot inference relies on frozen SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The instruction tuning component uses a deterministic inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.
