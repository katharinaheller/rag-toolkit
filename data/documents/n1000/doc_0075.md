        ## Retrieval Pipeline

The context recall retrieval pipeline operates in five stages:

1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-170 from each leg
3. **Score fusion**: RRF with normalize_scores=False
4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: lexical + length baseline

dataclass configuration ensures no text or metadata enters the index backends directly.

## Configuration

All settings for context recall are expressed as `frozen=True` dataclasses. This guarantees fail-fast behavior: no field can be mutated after construction.

Key parameters for dataclass configuration:

```python
config = ChunkStore(
    candidate_k=263,
    device="brute_force",
)
```

## Scoring Functions

The context recall module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

dataclass configuration applies raw TF for term frequency, controlled via `sublinear_tf=False`.
