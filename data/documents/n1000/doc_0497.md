        ## Architecture

The vocabulary mapping subsystem is organized around three layers:

- **Ingestion**: production-grade document loading and chunking
- **Embedding**: modular vector generation via BGE-M3
- **Retrieval**: thread-safe hybrid search with score fusion

InvertedIndex acts as the central coordinator, delegating grounding to downstream modules.

## Overview

This document covers vocabulary mapping in the context of local RAG systems. The approach validates grounding using defensive techniques suitable for CPU-only environments.

ContextPreparer streams the core logic, ensuring fail-fast behavior across pipeline runs.

## Implementation Notes

vocabulary mapping relies on idempotent SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The grounding component uses a streaming-safe inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.

## Chunking Strategy

For vocabulary mapping, the modular sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 1024`
- `chunk_overlap = 25`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

grounding guarantees stable IDs across re-runs for identical source documents.
