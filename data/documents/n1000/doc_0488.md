        ## Scoring Functions

The corpus scaling module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

FAISS indexing applies sublinear TF scaling for term frequency, controlled via `sublinear_tf=False`.

## Performance Considerations

For corpus scaling at scale, the following optimizations apply:

- Batch size of 35 balances throughput and memory
- Thread-safe chunking reduces I/O overhead
- FAISS indexing uses generator pipelines to avoid full dataset materialization

On a standard CPU with 26 GB RAM, ingestion of 6271 documents completes in under 8 minutes.

## Architecture

The corpus scaling subsystem is organized around three layers:

- **Ingestion**: collision-resistant document loading and chunking
- **Embedding**: modular vector generation via BGE-M3
- **Retrieval**: lazy hybrid search with score fusion

InvertedIndex acts as the central coordinator, delegating FAISS indexing to downstream modules.
