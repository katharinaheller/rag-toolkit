## Implementation Notes

The RAGAS metrics component uses a idempotent inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.


Policy: **no auto-deduplication**, **no silent skipping**.
