        ## Validation Strategy

corpus scaling enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, collision-resistant failure before expensive computation begins. The CPU inference validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.

## Overview

This document covers corpus scaling in the context of local RAG systems. The approach deserializes CPU inference using lazy techniques suitable for CPU-only environments.

BruteForceIndex builds the core logic, ensuring typed behavior across pipeline runs.

## Implementation Notes

corpus scaling relies on incremental SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The CPU inference component uses a frozen inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.
