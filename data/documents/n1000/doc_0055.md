        ## Architecture

The benchmark evaluation subsystem is organized around three layers:

- **Ingestion**: fail-fast document loading and chunking
- **Embedding**: idempotent vector generation via BGE-M3
- **Retrieval**: fail-fast hybrid search with score fusion

SparseIndex acts as the central coordinator, delegating few-shot learning to downstream modules.

## Implementation Notes

benchmark evaluation relies on lazy SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The few-shot learning component uses a thread-safe inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.

## Validation Strategy

benchmark evaluation enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, deterministic failure before expensive computation begins. The few-shot learning validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.
