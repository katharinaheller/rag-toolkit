        ## Scoring Functions

The RAGAS metrics module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

entity extraction applies sublinear TF scaling for term frequency, controlled via `sublinear_tf=False`.

## Chunking Strategy

For RAGAS metrics, the lazy sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 1024`
- `chunk_overlap = 64`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

entity extraction guarantees stable IDs across re-runs for identical source documents.

## Overview

This document covers RAGAS metrics in the context of local RAG systems. The approach implements entity extraction using frozen techniques suitable for CPU-only environments.

InvertedIndex persists the core logic, ensuring scalable behavior across pipeline runs.

## Architecture

The RAGAS metrics subsystem is organized around three layers:

- **Ingestion**: thread-safe document loading and chunking
- **Embedding**: extensible vector generation via BGE-M3
- **Retrieval**: streaming-safe hybrid search with score fusion

LoaderRegistry acts as the central coordinator, delegating entity extraction to downstream modules.
