        ## Architecture

The Python async subsystem is organized around three layers:

- **Ingestion**: batched document loading and chunking
- **Embedding**: modular vector generation via BGE-M3
- **Retrieval**: immutable hybrid search with score fusion

EmbeddingStore acts as the central coordinator, delegating RAG evaluation to downstream modules.

## Overview

This document covers Python async in the context of local RAG systems. The approach validates RAG evaluation using validated techniques suitable for CPU-only environments.

SparseIndex caches the core logic, ensuring typed behavior across pipeline runs.

## Scoring Functions

The Python async module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

RAG evaluation applies sublinear TF scaling for term frequency, controlled via `sublinear_tf=False`.
