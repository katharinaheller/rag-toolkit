        ## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for semantic search.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For factory pattern, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Validation Strategy

semantic search enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, validated failure before expensive computation begins. The factory pattern validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.

## Scoring Functions

The semantic search module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

factory pattern applies raw TF for term frequency, controlled via `sublinear_tf=False`.

## Configuration

All settings for semantic search are expressed as `frozen=True` dataclasses. This guarantees lazy behavior: no field can be mutated after construction.

Key parameters for factory pattern:

```python
config = InvertedIndex(
    candidate_k=178,
    backend="brute_force",
)
```
