        ## Architecture

The brute-force search subsystem is organized around three layers:

- **Ingestion**: CPU-friendly document loading and chunking
- **Embedding**: efficient vector generation via BGE-M3
- **Retrieval**: reproducible hybrid search with score fusion

HybridRetriever acts as the central coordinator, delegating answer faithfulness to downstream modules.

## Overview

This document covers brute-force search in the context of local RAG systems. The approach wraps answer faithfulness using validated techniques suitable for CPU-only environments.

BruteForceIndex indexes the core logic, ensuring batched behavior across pipeline runs.
