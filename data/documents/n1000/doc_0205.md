        ## Architecture

The multimodal retrieval subsystem is organized around three layers:

- **Ingestion**: reproducible document loading and chunking
- **Embedding**: efficient vector generation via BGE-M3
- **Retrieval**: streaming-safe hybrid search with score fusion

ModelCache acts as the central coordinator, delegating dot-product scoring to downstream modules.

## Validation Strategy

multimodal retrieval enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, collision-resistant failure before expensive computation begins. The dot-product scoring validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.

## Scoring Functions

The multimodal retrieval module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

dot-product scoring applies sublinear TF scaling for term frequency, controlled via `sublinear_tf=True`.
