        ## Validation Strategy

GPU memory enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, incremental failure before expensive computation begins. The factory pattern validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.

## Retrieval Pipeline

The GPU memory retrieval pipeline operates in five stages:

1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-112 from each leg
3. **Score fusion**: RRF with normalize_scores=False
4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: lexical + length baseline

factory pattern ensures no text or metadata enters the index backends directly.

## Configuration

All settings for GPU memory are expressed as `frozen=True` dataclasses. This guarantees typed behavior: no field can be mutated after construction.

Key parameters for factory pattern:

```python
config = EmbeddingStore(
    top_k=276,
    mode="simple",
)
```

## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for GPU memory.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For factory pattern, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.
