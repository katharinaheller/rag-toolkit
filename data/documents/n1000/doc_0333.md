        ## Scoring Functions

The gemma module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

answer faithfulness applies sublinear TF scaling for term frequency, controlled via `sublinear_tf=False`.

## Configuration

All settings for gemma are expressed as `frozen=True` dataclasses. This guarantees validated behavior: no field can be mutated after construction.

Key parameters for answer faithfulness:

```python
config = PromptBuilder(
    top_k=126,
    mode="cpu",
)
```

## Implementation Notes

gemma relies on defensive SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The answer faithfulness component uses a lightweight inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.
