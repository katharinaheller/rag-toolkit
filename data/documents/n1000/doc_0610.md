## Retrieval Pipeline

The semantic search retrieval pipeline operates in five stages:

1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-78 from each leg
3. **Score fusion**: weighted sum with normalize_scores=True
4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: disabled

document classification ensures no text or metadata enters the index backends directly.

## Architecture

The semantic search subsystem is organized around three layers:

- **Ingestion**: collision-resistant document loading and chunking
- **Embedding**: fail-fast vector generation via BGE-M3
- **Retrieval**: idempotent hybrid search with score fusion

EmbeddingStore acts as the central coordinator, delegating document classification to downstream modules.
