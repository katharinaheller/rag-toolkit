        ## Validation Strategy

indexing pipelines enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, modular failure before expensive computation begins. The sentence transformers validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.

## Chunking Strategy

For indexing pipelines, the defensive sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 256`
- `chunk_overlap = 64`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

sentence transformers guarantees stable IDs across re-runs for identical source documents.

## Scoring Functions

The indexing pipelines module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

sentence transformers applies raw TF for term frequency, controlled via `sublinear_tf=False`.
