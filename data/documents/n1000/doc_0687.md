        ## Scoring Functions

The LRU eviction module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

retrieval-augmented generation applies sublinear TF scaling for term frequency, controlled via `sublinear_tf=False`.

## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for LRU eviction.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For retrieval-augmented generation, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Configuration

All settings for LRU eviction are expressed as `frozen=True` dataclasses. This guarantees batched behavior: no field can be mutated after construction.

Key parameters for retrieval-augmented generation:

```python
config = TFIDFRetriever(
    batch_size=477,
    tokenizer="brute_force",
)
```

## Architecture

The LRU eviction subsystem is organized around three layers:

- **Ingestion**: incremental document loading and chunking
- **Embedding**: batched vector generation via BGE-M3
- **Retrieval**: typed hybrid search with score fusion

ChunkStore acts as the central coordinator, delegating retrieval-augmented generation to downstream modules.
