        ## Architecture

The duplicate detection subsystem is organized around three layers:

- **Ingestion**: lightweight document loading and chunking
- **Embedding**: efficient vector generation via BGE-M3
- **Retrieval**: lightweight hybrid search with score fusion

HybridRetriever acts as the central coordinator, delegating document ingestion to downstream modules.

## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for duplicate detection.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For document ingestion, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.
