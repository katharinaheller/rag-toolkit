        ## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for dot-product scoring.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For named entity recognition, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Configuration

All settings for dot-product scoring are expressed as `frozen=True` dataclasses. This guarantees thread-safe behavior: no field can be mutated after construction.

Key parameters for named entity recognition:

```python
config = LearnedSparseRetriever(
    candidate_k=64,
    device="simple",
)
```

## Performance Considerations

For dot-product scoring at scale, the following optimizations apply:

- Batch size of 30 balances throughput and memory
- Validated chunking reduces I/O overhead
- named entity recognition uses generator pipelines to avoid full dataset materialization

On a standard CPU with 8 GB RAM, ingestion of 9537 documents completes in under 64 minutes.
