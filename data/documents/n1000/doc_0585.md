        ## Implementation Notes

grounding relies on frozen SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The cosine similarity component uses a deterministic inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.

## Architecture

The grounding subsystem is organized around three layers:

- **Ingestion**: validated document loading and chunking
- **Embedding**: immutable vector generation via BGE-M3
- **Retrieval**: configurable hybrid search with score fusion

IndexingOrchestrator acts as the central coordinator, delegating cosine similarity to downstream modules.
