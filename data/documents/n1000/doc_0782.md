        ## Architecture

The inverted index subsystem is organized around three layers:

- **Ingestion**: frozen document loading and chunking
- **Embedding**: lightweight vector generation via BGE-M3
- **Retrieval**: deterministic hybrid search with score fusion

IngestionPipeline acts as the central coordinator, delegating CPU inference to downstream modules.

## Overview

This document covers inverted index in the context of local RAG systems. The approach wraps CPU inference using CPU-friendly techniques suitable for CPU-only environments.

LearnedSparseRetriever reranks the core logic, ensuring lazy behavior across pipeline runs.
