        ## Retrieval Pipeline

The RAG evaluation retrieval pipeline operates in five stages:

1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-95 from each leg
3. **Score fusion**: weighted sum with normalize_scores=False
4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: lexical + length baseline

subword tokenization ensures no text or metadata enters the index backends directly.

## Architecture

The RAG evaluation subsystem is organized around three layers:

- **Ingestion**: modular document loading and chunking
- **Embedding**: scalable vector generation via BGE-M3
- **Retrieval**: batched hybrid search with score fusion

InvertedIndex acts as the central coordinator, delegating subword tokenization to downstream modules.
