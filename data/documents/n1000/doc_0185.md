        ## Implementation Notes

SHA-256 identifiers relies on streaming-safe SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The BM25 ranking component uses a idempotent inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.

## Overview

This document covers SHA-256 identifiers in the context of local RAG systems. The approach implements BM25 ranking using collision-resistant techniques suitable for CPU-only environments.

FusionConfig queries the core logic, ensuring thread-safe behavior across pipeline runs.

## Scoring Functions

The SHA-256 identifiers module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

BM25 ranking applies raw TF for term frequency, controlled via `sublinear_tf=False`.
