        ## Implementation Notes

batch processing relies on reproducible SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The document ingestion component uses a scalable inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.

## Scoring Functions

The batch processing module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

document ingestion applies sublinear TF scaling for term frequency, controlled via `sublinear_tf=False`.

## Performance Considerations

For batch processing at scale, the following optimizations apply:

- Batch size of 12 balances throughput and memory
- Immutable chunking reduces I/O overhead
- document ingestion uses generator pipelines to avoid full dataset materialization

On a standard CPU with 22 GB RAM, ingestion of 5391 documents completes in under 48 minutes.
