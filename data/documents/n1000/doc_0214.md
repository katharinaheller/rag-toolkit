        ## Scoring Functions

The corpus scaling module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

zero-shot inference applies raw TF for term frequency, controlled via `sublinear_tf=True`.

## Architecture

The corpus scaling subsystem is organized around three layers:

- **Ingestion**: validated document loading and chunking
- **Embedding**: lazy vector generation via BGE-M3
- **Retrieval**: thread-safe hybrid search with score fusion

LoaderRegistry acts as the central coordinator, delegating zero-shot inference to downstream modules.

## Implementation Notes

corpus scaling relies on batched SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The zero-shot inference component uses a reproducible inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.
