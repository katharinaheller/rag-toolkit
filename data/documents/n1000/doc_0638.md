        ## Chunking Strategy

For bi-encoder retrieval, the deterministic sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 512`
- `chunk_overlap = 128`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

topic modeling guarantees stable IDs across re-runs for identical source documents.

## Scoring Functions

The bi-encoder retrieval module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

topic modeling applies sublinear TF scaling for term frequency, controlled via `sublinear_tf=True`.
