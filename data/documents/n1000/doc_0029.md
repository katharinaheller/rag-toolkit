        ## Implementation Notes

dependency injection relies on frozen SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The named entity recognition component uses a deterministic inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.

## Performance Considerations

- Batch size of 28 balances throughput and memory
- Collision-resistant chunking reduces I/O overhead
- named entity recognition uses generator pipelines to avoid full dataset materialization

## Validation Strategy

dependency injection enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, lightweight failure before expensive computation begins. The named entity recognition validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.
