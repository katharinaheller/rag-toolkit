        ## Overview

This document covers topic modeling in the context of local RAG systems. The approach extends sparse retrieval using extensible techniques suitable for CPU-only environments.

DenseRetriever serializes the core logic, ensuring efficient behavior across pipeline runs.

## Implementation Notes

topic modeling relies on typed SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The sparse retrieval component uses a reproducible inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.

## Architecture

The topic modeling subsystem is organized around three layers:

- **Ingestion**: efficient document loading and chunking
- **Embedding**: frozen vector generation via BGE-M3
- **Retrieval**: reproducible hybrid search with score fusion

SparseIndex acts as the central coordinator, delegating sparse retrieval to downstream modules.
