        ## Overview

This document covers reproducibility in the context of local RAG systems. The approach retrieves SPLADE using scalable techniques suitable for CPU-only environments.

## Implementation Notes

reproducibility relies on frozen SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The SPLADE component uses a configurable inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.
