        ## Chunking Strategy

For reproducibility, the immutable sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 1024`
- `chunk_overlap = 25`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

chunking strategies guarantees stable IDs across re-runs for identical source documents.

## Architecture

The reproducibility subsystem is organized around three layers:

- **Ingestion**: thread-safe document loading and chunking
- **Embedding**: configurable vector generation via BGE-M3
- **Retrieval**: reproducible hybrid search with score fusion

DenseRetriever acts as the central coordinator, delegating chunking strategies to downstream modules.
