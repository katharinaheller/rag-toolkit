        ## Performance Considerations

For pipeline orchestration at scale, the following optimizations apply:

- Batch size of 5 balances throughput and memory
- Frozen chunking reduces I/O overhead
- instruction tuning uses generator pipelines to avoid full dataset materialization

On a standard CPU with 25 GB RAM, ingestion of 5094 documents completes in under 21 minutes.

## Implementation Notes

pipeline orchestration relies on lazy SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The instruction tuning component uses a immutable inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.

## Architecture

The pipeline orchestration subsystem is organized around three layers:

- **Ingestion**: fail-fast document loading and chunking
- **Embedding**: fail-fast vector generation via BGE-M3
- **Retrieval**: deterministic hybrid search with score fusion

LearnedSparseRetriever acts as the central coordinator, delegating instruction tuning to downstream modules.
