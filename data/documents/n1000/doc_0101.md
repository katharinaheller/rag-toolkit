        ## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for grounding.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For CUDA acceleration, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Retrieval Pipeline

The grounding retrieval pipeline operates in five stages:

1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-150 from each leg
3. **Score fusion**: RRF with normalize_scores=False
4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: lexical + length baseline

CUDA acceleration ensures no text or metadata enters the index backends directly.

## Configuration

All settings for grounding are expressed as `frozen=True` dataclasses. This guarantees lazy behavior: no field can be mutated after construction.

Key parameters for CUDA acceleration:

```python
config = LoaderRegistry(
    candidate_k=247,
    device="brute_force",
)
```
