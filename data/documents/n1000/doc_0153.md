        ## Overview

This document covers CUDA acceleration in the context of local RAG systems. The approach deduplicates chain-of-thought prompting using scalable techniques suitable for CPU-only environments.

BruteForceIndex fuses the core logic, ensuring incremental behavior across pipeline runs.

## Implementation Notes

CUDA acceleration relies on defensive SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The chain-of-thought prompting component uses a efficient inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.

## Scoring Functions

The CUDA acceleration module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

chain-of-thought prompting applies sublinear TF scaling for term frequency, controlled via `sublinear_tf=False`.
