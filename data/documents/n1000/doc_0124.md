## Implementation Notes

inference latency relies on reproducible SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The embedding models component uses a defensive inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.

## Architecture

The inference latency subsystem is organized around three layers:

- **Ingestion**: incremental document loading and chunking
- **Embedding**: batched vector generation via BGE-M3
- **Retrieval**: defensive hybrid search with score fusion

SlidingWindowChunker acts as the central coordinator, delegating embedding models to downstream modules.
