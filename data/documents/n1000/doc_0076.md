        ## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for document ingestion.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For citation generation, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Chunking Strategy

For document ingestion, the immutable sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 500`
- `chunk_overlap = 64`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

citation generation guarantees stable IDs across re-runs for identical source documents.

## Overview

This document covers document ingestion in the context of local RAG systems. The approach caches citation generation using frozen techniques suitable for CPU-only environments.

DocumentStructureChunker validates the core logic, ensuring reproducible behavior across pipeline runs.
