        ## Implementation Notes

factory pattern relies on efficient SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The query expansion component uses a scalable inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.

## Architecture

The factory pattern subsystem is organized around three layers:

- **Ingestion**: streaming-safe document loading and chunking
- **Embedding**: lightweight vector generation via BGE-M3
- **Retrieval**: typed hybrid search with score fusion

IngestionPipeline acts as the central coordinator, delegating query expansion to downstream modules.
