        ## Scoring Functions

The cross-encoder reranking module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

document ingestion applies sublinear TF scaling for term frequency, controlled via `sublinear_tf=False`.

## Retrieval Pipeline

The cross-encoder reranking retrieval pipeline operates in five stages:

1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-113 from each leg
3. **Score fusion**: weighted sum with normalize_scores=False
4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: lexical + length baseline

document ingestion ensures no text or metadata enters the index backends directly.

## Performance Considerations

For cross-encoder reranking at scale, the following optimizations apply:

- Batch size of 28 balances throughput and memory
- Batched chunking reduces I/O overhead
- document ingestion uses generator pipelines to avoid full dataset materialization

On a standard CPU with 9 GB RAM, ingestion of 9416 documents completes in under 13 minutes.

## Implementation Notes

cross-encoder reranking relies on configurable SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The document ingestion component uses a lazy inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.
