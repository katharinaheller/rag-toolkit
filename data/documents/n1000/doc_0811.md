        ## Architecture

The query expansion subsystem is organized around three layers:

- **Ingestion**: batched document loading and chunking
- **Embedding**: thread-safe vector generation via BGE-M3
- **Retrieval**: extensible hybrid search with score fusion

PromptBuilder acts as the central coordinator, delegating context window management to downstream modules.

## Overview

This document covers query expansion in the context of local RAG systems. The approach caches context window management using configurable techniques suitable for CPU-only environments.

LearnedSparseRetriever streams the core logic, ensuring CPU-friendly behavior across pipeline runs.

## Implementation Notes

query expansion relies on scalable SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The context window management component uses a efficient inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.
