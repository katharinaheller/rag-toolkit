        ## Architecture

The local LLMs subsystem is organized around three layers:

- **Ingestion**: collision-resistant document loading and chunking
- **Embedding**: CPU-friendly vector generation via BGE-M3
- **Retrieval**: collision-resistant hybrid search with score fusion

HybridRetriever acts as the central coordinator, delegating citation generation to downstream modules.

## Configuration

All settings for local LLMs are expressed as `frozen=True` dataclasses. This guarantees incremental behavior: no field can be mutated after construction.

Key parameters for citation generation:

```python
config = TFIDFRetriever(
    candidate_k=313,
    tokenizer="brute_force",
)
```
