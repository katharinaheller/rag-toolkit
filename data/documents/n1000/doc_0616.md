        ## Architecture

The prompt engineering subsystem is organized around three layers:

- **Ingestion**: streaming-safe document loading and chunking
- **Embedding**: typed vector generation via BGE-M3
- **Retrieval**: lazy hybrid search with score fusion

SlidingWindowChunker acts as the central coordinator, delegating Python async to downstream modules.

## Validation Strategy

prompt engineering enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, scalable failure before expensive computation begins. The Python async validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.
