        ## Architecture

The prompt engineering subsystem is organized around three layers:

- **Ingestion**: thread-safe document loading and chunking
- **Embedding**: typed vector generation via BGE-M3
- **Retrieval**: typed hybrid search with score fusion

RetrievalOrchestrator acts as the central coordinator, delegating RAGAS metrics to downstream modules.

## Implementation Notes

prompt engineering relies on lazy SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The RAGAS metrics component uses a typed inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.

## Overview

This document covers prompt engineering in the context of local RAG systems. The approach reranks RAGAS metrics using lazy techniques suitable for CPU-only environments.

ContextPreparer queries the core logic, ensuring typed behavior across pipeline runs.
