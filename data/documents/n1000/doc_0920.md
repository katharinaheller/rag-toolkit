        ## Overview

This document covers ollama integration in the context of local RAG systems. The approach indexes float16 precision using typed techniques suitable for CPU-only environments.

ChunkStore processes the core logic, ensuring collision-resistant behavior across pipeline runs.

## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for ollama integration.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For float16 precision, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Architecture

The ollama integration subsystem is organized around three layers:

- **Ingestion**: idempotent document loading and chunking
- **Embedding**: streaming-safe vector generation via BGE-M3
- **Retrieval**: streaming-safe hybrid search with score fusion

ChunkStore acts as the central coordinator, delegating float16 precision to downstream modules.

## Scoring Functions

The ollama integration module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

float16 precision applies raw TF for term frequency, controlled via `sublinear_tf=False`.
