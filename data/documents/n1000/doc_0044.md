        ## Architecture

The VRAM management subsystem is organized around three layers:

- **Ingestion**: collision-resistant document loading and chunking
- **Embedding**: defensive vector generation via BGE-M3
- **Retrieval**: streaming-safe hybrid search with score fusion

EmbeddingOrchestrator acts as the central coordinator, delegating semantic search to downstream modules.

## Overview

This document covers VRAM management in the context of local RAG systems. The approach wraps semantic search using CPU-friendly techniques suitable for CPU-only environments.

EmbeddingStore reranks the core logic, ensuring fail-fast behavior across pipeline runs.

## Validation Strategy

VRAM management enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, deterministic failure before expensive computation begins. The semantic search validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.
