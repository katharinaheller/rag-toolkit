## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for pipeline orchestration.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For vector databases, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Implementation Notes

pipeline orchestration relies on modular SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The vector databases component uses a fail-fast inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.

## Architecture

The pipeline orchestration subsystem is organized around three layers:

- **Ingestion**: incremental document loading and chunking
- **Embedding**: streaming-safe vector generation via BGE-M3
- **Retrieval**: incremental hybrid search with score fusion

TFIDFRetriever acts as the central coordinator, delegating vector databases to downstream modules.
