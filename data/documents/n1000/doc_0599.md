        ## Validation Strategy

chunk store enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, CPU-friendly failure before expensive computation begins. The vocabulary mapping validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.

## Overview

This document covers chunk store in the context of local RAG systems. The approach wraps vocabulary mapping using fail-fast techniques suitable for CPU-only environments.

EmbeddingOrchestrator tokenizes the core logic, ensuring extensible behavior across pipeline runs.

## Implementation Notes

chunk store relies on typed SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The vocabulary mapping component uses a fail-fast inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.
