        ## Architecture

The model quantization subsystem is organized around three layers:

- **Ingestion**: thread-safe document loading and chunking
- **Embedding**: incremental vector generation via BGE-M3
- **Retrieval**: lazy hybrid search with score fusion

EmbeddingStore acts as the central coordinator, delegating topic modeling to downstream modules.

## Retrieval Pipeline

The model quantization retrieval pipeline operates in five stages:

1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-82 from each leg
3. **Score fusion**: weighted sum with normalize_scores=False
4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: lexical + length baseline

topic modeling ensures no text or metadata enters the index backends directly.
