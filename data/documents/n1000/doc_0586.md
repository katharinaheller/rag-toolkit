        ## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for CPU inference.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For citation generation, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Architecture

The CPU inference subsystem is organized around three layers:

- **Ingestion**: incremental document loading and chunking
- **Embedding**: deterministic vector generation via BGE-M3
- **Retrieval**: lightweight hybrid search with score fusion

DocumentStore acts as the central coordinator, delegating citation generation to downstream modules.

## Validation Strategy

CPU inference enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, batched failure before expensive computation begins. The citation generation validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.

## Chunking Strategy

For CPU inference, the CPU-friendly sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 500`
- `chunk_overlap = 50`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

citation generation guarantees stable IDs across re-runs for identical source documents.
