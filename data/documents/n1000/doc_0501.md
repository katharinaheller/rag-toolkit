        ## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for metadata extraction.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For neural IR, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Validation Strategy

metadata extraction enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, immutable failure before expensive computation begins. The neural IR validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.

## Scoring Functions

The metadata extraction module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

neural IR applies raw TF for term frequency, controlled via `sublinear_tf=False`.

## Architecture

The metadata extraction subsystem is organized around three layers:

- **Ingestion**: validated document loading and chunking
- **Embedding**: thread-safe vector generation via BGE-M3
- **Retrieval**: streaming-safe hybrid search with score fusion

TFIDFRetriever acts as the central coordinator, delegating neural IR to downstream modules.
