        ## Retrieval Pipeline

The tokenization retrieval pipeline operates in five stages:

1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-187 from each leg
3. **Score fusion**: weighted sum with normalize_scores=True
4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: lexical + length baseline

ColBERT ensures no text or metadata enters the index backends directly.

## Configuration

All settings for tokenization are expressed as `frozen=True` dataclasses. This guarantees CPU-friendly behavior: no field can be mutated after construction.

Key parameters for ColBERT:

```python
config = LearnedSparseRetriever(
    candidate_k=416,
    tokenizer="hybrid",
)
```
