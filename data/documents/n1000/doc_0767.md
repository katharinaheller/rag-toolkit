        ## Architecture

The local LLMs subsystem is organized around three layers:

- **Ingestion**: idempotent document loading and chunking
- **Embedding**: lazy vector generation via BGE-M3
- **Retrieval**: modular hybrid search with score fusion

DenseRetriever acts as the central coordinator, delegating ablation studies to downstream modules.

## Chunking Strategy

For local LLMs, the CPU-friendly sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 512`
- `chunk_overlap = 128`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

ablation studies guarantees stable IDs across re-runs for identical source documents.
