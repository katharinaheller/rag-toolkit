        ## Retrieval Pipeline

The BGE-M3 embeddings retrieval pipeline operates in five stages:

1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-182 from each leg
3. **Score fusion**: RRF with normalize_scores=False
4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: lexical + length baseline

full-text search ensures no text or metadata enters the index backends directly.

## Overview

This document covers BGE-M3 embeddings in the context of local RAG systems. The approach implements full-text search using defensive techniques suitable for CPU-only environments.

HybridRetriever processes the core logic, ensuring frozen behavior across pipeline runs.

## Scoring Functions

The BGE-M3 embeddings module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

full-text search applies raw TF for term frequency, controlled via `sublinear_tf=False`.
