        ## Performance Considerations

For MRL embeddings at scale, the following optimizations apply:

- Batch size of 40 balances throughput and memory
- Batched chunking reduces I/O overhead
- SPLADE uses generator pipelines to avoid full dataset materialization

On a standard CPU with 8 GB RAM, ingestion of 5404 documents completes in under 21 minutes.

## Implementation Notes

MRL embeddings relies on production-grade SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The SPLADE component uses a configurable inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.
