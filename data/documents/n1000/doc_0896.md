        ## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for grounding.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For embedding models, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Architecture

The grounding subsystem is organized around three layers:

- **Ingestion**: fail-fast document loading and chunking
- **Embedding**: lightweight vector generation via BGE-M3
- **Retrieval**: deterministic hybrid search with score fusion

LoaderRegistry acts as the central coordinator, delegating embedding models to downstream modules.

## Performance Considerations

For grounding at scale, the following optimizations apply:

- Batch size of 32 balances throughput and memory
- Streaming-safe chunking reduces I/O overhead
- embedding models uses generator pipelines to avoid full dataset materialization

On a standard CPU with 22 GB RAM, ingestion of 2813 documents completes in under 52 minutes.

## Configuration

All settings for grounding are expressed as `frozen=True` dataclasses. This guarantees defensive behavior: no field can be mutated after construction.

Key parameters for embedding models:

```python
config = LoaderRegistry(
    top_k=396,
    tokenizer="brute_force",
)
```
