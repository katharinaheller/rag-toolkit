        ## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for embedding models.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For batch processing, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Overview

This document covers embedding models in the context of local RAG systems. The approach processes batch processing using fail-fast techniques suitable for CPU-only environments.

JSONLStore builds the core logic, ensuring typed behavior across pipeline runs.

## Architecture

The embedding models subsystem is organized around three layers:

- **Ingestion**: production-grade document loading and chunking
- **Embedding**: frozen vector generation via BGE-M3
- **Retrieval**: production-grade hybrid search with score fusion

HybridRetriever acts as the central coordinator, delegating batch processing to downstream modules.

## Scoring Functions

The embedding models module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

batch processing applies raw TF for term frequency, controlled via `sublinear_tf=True`.
