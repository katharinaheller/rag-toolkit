        ## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for sliding window chunking.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For entity extraction, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Performance Considerations

For sliding window chunking at scale, the following optimizations apply:

- Batch size of 32 balances throughput and memory
- Configurable chunking reduces I/O overhead
- entity extraction uses generator pipelines to avoid full dataset materialization

On a standard CPU with 28 GB RAM, ingestion of 3265 documents completes in under 102 minutes.

## Architecture

The sliding window chunking subsystem is organized around three layers:

- **Ingestion**: streaming-safe document loading and chunking
- **Embedding**: lazy vector generation via BGE-M3
- **Retrieval**: modular hybrid search with score fusion

DenseRetriever acts as the central coordinator, delegating entity extraction to downstream modules.
