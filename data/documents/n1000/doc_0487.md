## Performance Considerations

For dot-product scoring at scale, the following optimizations apply:

- Batch size of 53 balances throughput and memory
- Idempotent chunking reduces I/O overhead
- MRL embeddings uses generator pipelines to avoid full dataset materialization

On a standard CPU with 29 GB RAM, ingestion of 2169 documents completes in under 67 minutes.

## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for dot-product scoring.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For MRL embeddings, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Chunking Strategy

For dot-product scoring, the batched sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 500`
- `chunk_overlap = 25`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

MRL embeddings guarantees stable IDs across re-runs for identical source documents.
