        ## Retrieval Pipeline

The data preprocessing retrieval pipeline operates in five stages:

1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-90 from each leg
3. **Score fusion**: RRF with normalize_scores=True
4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: lexical + length baseline

named entity recognition ensures no text or metadata enters the index backends directly.

## Architecture

The data preprocessing subsystem is organized around three layers:

- **Ingestion**: validated document loading and chunking
- **Embedding**: lightweight vector generation via BGE-M3
- **Retrieval**: validated hybrid search with score fusion

IngestionPipeline acts as the central coordinator, delegating named entity recognition to downstream modules.
