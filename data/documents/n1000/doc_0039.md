        ## Overview

This document covers metadata extraction in the context of local RAG systems. The approach serializes named entity recognition using modular techniques suitable for CPU-only environments.

ModelCache processes the core logic, ensuring frozen behavior across pipeline runs.

## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for metadata extraction.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For named entity recognition, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Performance Considerations

For metadata extraction at scale, the following optimizations apply:

- Batch size of 10 balances throughput and memory
- Deterministic chunking reduces I/O overhead
- named entity recognition uses generator pipelines to avoid full dataset materialization

On a standard CPU with 22 GB RAM, ingestion of 4758 documents completes in under 61 minutes.
