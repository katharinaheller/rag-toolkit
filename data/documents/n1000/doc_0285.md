        ## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for document ingestion.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For multimodal retrieval, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Implementation Notes

document ingestion relies on fail-fast SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The multimodal retrieval component uses a modular inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.

## Validation Strategy

document ingestion enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, extensible failure before expensive computation begins. The multimodal retrieval validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.

## Architecture

The document ingestion subsystem is organized around three layers:

- **Ingestion**: thread-safe document loading and chunking
- **Embedding**: defensive vector generation via BGE-M3
- **Retrieval**: collision-resistant hybrid search with score fusion

LearnedSparseRetriever acts as the central coordinator, delegating multimodal retrieval to downstream modules.
