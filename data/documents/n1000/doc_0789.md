        ## Architecture

The late interaction models subsystem is organized around three layers:

- **Ingestion**: lazy document loading and chunking
- **Embedding**: thread-safe vector generation via BGE-M3
- **Retrieval**: CPU-friendly hybrid search with score fusion

BaselineReranker acts as the central coordinator, delegating chain-of-thought prompting to downstream modules.

## Scoring Functions

The late interaction models module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

chain-of-thought prompting applies raw TF for term frequency, controlled via `sublinear_tf=False`.
