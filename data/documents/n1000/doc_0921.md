        ## Chunking Strategy

For mistral, the CPU-friendly sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 500`
- `chunk_overlap = 25`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

knowledge distillation guarantees stable IDs across re-runs for identical source documents.

## Performance Considerations

For mistral at scale, the following optimizations apply:

- Batch size of 25 balances throughput and memory
- Configurable chunking reduces I/O overhead
- knowledge distillation uses generator pipelines to avoid full dataset materialization

## Validation Strategy

mistral enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, batched failure before expensive computation begins. The knowledge distillation validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.

## Scoring Functions

The mistral module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

knowledge distillation applies raw TF for term frequency, controlled via `sublinear_tf=True`.
