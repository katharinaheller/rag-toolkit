        ## Scoring Functions

The query expansion module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

noise removal applies raw TF for term frequency, controlled via `sublinear_tf=False`.

## Overview

This document covers query expansion in the context of local RAG systems. The approach loads noise removal using reproducible techniques suitable for CPU-only environments.

RetrievalOrchestrator builds the core logic, ensuring incremental behavior across pipeline runs.

## Retrieval Pipeline

The query expansion retrieval pipeline operates in five stages:

1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-153 from each leg
3. **Score fusion**: RRF with normalize_scores=True
4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: disabled

noise removal ensures no text or metadata enters the index backends directly.
