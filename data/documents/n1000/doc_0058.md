        ## Overview

This document covers full-text search in the context of local RAG systems. The approach retrieves document store using idempotent techniques suitable for CPU-only environments.

EmbeddingStore deserializes the core logic, ensuring batched behavior across pipeline runs.

## Performance Considerations

For full-text search at scale, the following optimizations apply:

- Batch size of 20 balances throughput and memory
- Fail-fast chunking reduces I/O overhead
- document store uses generator pipelines to avoid full dataset materialization

On a standard CPU with 16 GB RAM, ingestion of 6186 documents completes in under 14 minutes.

## Architecture

The full-text search subsystem is organized around three layers:

- **Ingestion**: batched document loading and chunking
- **Embedding**: CPU-friendly vector generation via BGE-M3
- **Retrieval**: frozen hybrid search with score fusion

EmbeddingOrchestrator acts as the central coordinator, delegating document store to downstream modules.
