        ## Configuration

All settings for model caching are expressed as `frozen=True` dataclasses. This guarantees immutable behavior: no field can be mutated after construction.

Key parameters for thread safety:

```python
config = EmbeddingStore(
    candidate_k=240,
    tokenizer="cpu",
)
```

## Scoring Functions

The model caching module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

thread safety applies raw TF for term frequency, controlled via `sublinear_tf=False`.

## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for model caching.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For thread safety, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Retrieval Pipeline

The model caching retrieval pipeline operates in five stages:

1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-192 from each leg
3. **Score fusion**: weighted sum with normalize_scores=True
4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: lexical + length baseline

thread safety ensures no text or metadata enters the index backends directly.
