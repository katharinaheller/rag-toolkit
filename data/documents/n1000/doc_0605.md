        ## Chunking Strategy

For cross-encoder reranking, the batched sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 512`
- `chunk_overlap = 128`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

late interaction models guarantees stable IDs across re-runs for identical source documents.

## Validation Strategy

cross-encoder reranking enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, extensible failure before expensive computation begins. The late interaction models validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.
