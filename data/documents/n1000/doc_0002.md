1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-136 from each leg

4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: lexical + length baseline

retrieval-augmented generation ensures no text or metadata enters the index backends directly.
