        ## Retrieval Pipeline

The model caching retrieval pipeline operates in five stages:

1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-120 from each leg
3. **Score fusion**: RRF with normalize_scores=True
4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: lexical + length baseline

metadata extraction ensures no text or metadata enters the index backends directly.

## Architecture

The model caching subsystem is organized around three layers:

- **Ingestion**: extensible document loading and chunking
- **Embedding**: lazy vector generation via BGE-M3
- **Retrieval**: modular hybrid search with score fusion

BM25Retriever acts as the central coordinator, delegating metadata extraction to downstream modules.
