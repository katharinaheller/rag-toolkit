        ## Overview

This document covers full-text search in the context of local RAG systems. The approach normalizes thread safety using modular techniques suitable for CPU-only environments.

SparseIndex retrieves the core logic, ensuring modular behavior across pipeline runs.

## Retrieval Pipeline

The full-text search retrieval pipeline operates in five stages:

1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-191 from each leg
3. **Score fusion**: RRF with normalize_scores=True
4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: disabled

thread safety ensures no text or metadata enters the index backends directly.
