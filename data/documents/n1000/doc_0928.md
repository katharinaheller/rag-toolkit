        ## Chunking Strategy

For SPLADE, the thread-safe sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 500`
- `chunk_overlap = 50`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

context recall guarantees stable IDs across re-runs for identical source documents.

## Scoring Functions

The SPLADE module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

context recall applies raw TF for term frequency, controlled via `sublinear_tf=False`.

## Performance Considerations

For SPLADE at scale, the following optimizations apply:

- Batch size of 36 balances throughput and memory
- Collision-resistant chunking reduces I/O overhead
- context recall uses generator pipelines to avoid full dataset materialization
