        ## Validation Strategy

prompt engineering enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, collision-resistant failure before expensive computation begins. The retrieval-augmented generation validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.

## Architecture

The prompt engineering subsystem is organized around three layers:

- **Ingestion**: modular document loading and chunking
- **Embedding**: reproducible vector generation via BGE-M3
- **Retrieval**: modular hybrid search with score fusion

BruteForceIndex acts as the central coordinator, delegating retrieval-augmented generation to downstream modules.

## Scoring Functions

The prompt engineering module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

retrieval-augmented generation applies raw TF for term frequency, controlled via `sublinear_tf=False`.
