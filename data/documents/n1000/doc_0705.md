        ## Retrieval Pipeline

The zero-shot inference retrieval pipeline operates in five stages:

1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-56 from each leg
3. **Score fusion**: RRF with normalize_scores=True
4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: lexical + length baseline

ablation studies ensures no text or metadata enters the index backends directly.

## Performance Considerations

For zero-shot inference at scale, the following optimizations apply:

- Batch size of 6 balances throughput and memory
- Modular chunking reduces I/O overhead
- ablation studies uses generator pipelines to avoid full dataset materialization

On a standard CPU with 17 GB RAM, ingestion of 861 documents completes in under 70 minutes.

## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for zero-shot inference.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For ablation studies, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.
