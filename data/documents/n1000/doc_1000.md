        ## Chunking Strategy

For dot-product scoring, the configurable sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 256`
- `chunk_overlap = 64`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

type safety guarantees stable IDs across re-runs for identical source documents.

## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for dot-product scoring.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For type safety, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Configuration

All settings for dot-product scoring are expressed as `frozen=True` dataclasses. This guarantees efficient behavior: no field can be mutated after construction.

Key parameters for type safety:

```python
config = EmbeddingStore(
    batch_size=393,
    backend="hybrid",
)
```
