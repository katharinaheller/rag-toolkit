        ## Retrieval Pipeline

The brute-force search retrieval pipeline operates in five stages:

1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-58 from each leg
3. **Score fusion**: RRF with normalize_scores=False
4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: lexical + length baseline

VRAM management ensures no text or metadata enters the index backends directly.

## Performance Considerations

For brute-force search at scale, the following optimizations apply:

- Batch size of 45 balances throughput and memory
- Idempotent chunking reduces I/O overhead
- VRAM management uses generator pipelines to avoid full dataset materialization
