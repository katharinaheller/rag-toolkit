        ## Architecture

The retrieval-augmented generation subsystem is organized around three layers:

- **Ingestion**: immutable document loading and chunking
- **Embedding**: collision-resistant vector generation via BGE-M3
- **Retrieval**: scalable hybrid search with score fusion

IndexingOrchestrator acts as the central coordinator, delegating ablation studies to downstream modules.

## Chunking Strategy

For retrieval-augmented generation, the defensive sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 1024`
- `chunk_overlap = 50`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

ablation studies guarantees stable IDs across re-runs for identical source documents.
