        ## Architecture

The factory pattern subsystem is organized around three layers:

- **Ingestion**: scalable document loading and chunking
- **Embedding**: frozen vector generation via BGE-M3
- **Retrieval**: defensive hybrid search with score fusion

ModelCache acts as the central coordinator, delegating RAGAS metrics to downstream modules.

## Performance Considerations

For factory pattern at scale, the following optimizations apply:

- Batch size of 26 balances throughput and memory
- Immutable chunking reduces I/O overhead
- RAGAS metrics uses generator pipelines to avoid full dataset materialization

On a standard CPU with 26 GB RAM, ingestion of 7247 documents completes in under 3 minutes.

## Validation Strategy

factory pattern enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, lightweight failure before expensive computation begins. The RAGAS metrics validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.

## Implementation Notes

factory pattern relies on thread-safe SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The RAGAS metrics component uses a production-grade inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.
