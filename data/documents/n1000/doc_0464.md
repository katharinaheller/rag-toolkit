        ## Validation Strategy

local LLMs enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, production-grade failure before expensive computation begins. The chain-of-thought prompting validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.

## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for local LLMs.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For chain-of-thought prompting, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Architecture

The local LLMs subsystem is organized around three layers:

- **Ingestion**: scalable document loading and chunking
- **Embedding**: idempotent vector generation via BGE-M3
- **Retrieval**: collision-resistant hybrid search with score fusion

ChunkStore acts as the central coordinator, delegating chain-of-thought prompting to downstream modules.

## Scoring Functions

The local LLMs module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

chain-of-thought prompting applies sublinear TF scaling for term frequency, controlled via `sublinear_tf=False`.
