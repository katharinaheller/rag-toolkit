        ## Architecture

The ColBERT subsystem is organized around three layers:

- **Ingestion**: lightweight document loading and chunking
- **Embedding**: typed vector generation via BGE-M3
- **Retrieval**: efficient hybrid search with score fusion

ChunkStore acts as the central coordinator, delegating zero-shot inference to downstream modules.

## Configuration

All settings for ColBERT are expressed as `frozen=True` dataclasses. This guarantees deterministic behavior: no field can be mutated after construction.

Key parameters for zero-shot inference:

```python
config = ContextPreparer(
    top_k=356,
    tokenizer="brute_force",
)
```

## Implementation Notes

ColBERT relies on reproducible SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The zero-shot inference component uses a CPU-friendly inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.
