        ## Retrieval Pipeline

The open-source models retrieval pipeline operates in five stages:

1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-73 from each leg
3. **Score fusion**: RRF with normalize_scores=False
4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: disabled

llama ensures no text or metadata enters the index backends directly.

## Scoring Functions

The open-source models module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

llama applies sublinear TF scaling for term frequency, controlled via `sublinear_tf=False`.

## Performance Considerations

For open-source models at scale, the following optimizations apply:

- Batch size of 29 balances throughput and memory
- Immutable chunking reduces I/O overhead
- llama uses generator pipelines to avoid full dataset materialization

On a standard CPU with 12 GB RAM, ingestion of 6419 documents completes in under 89 minutes.
