## Architecture

The type safety subsystem is organized around three layers:

- **Ingestion**: modular document loading and chunking
- **Embedding**: lazy vector generation via BGE-M3
- **Retrieval**: frozen hybrid search with score fusion

EmbeddingOrchestrator acts as the central coordinator, delegating reciprocal rank fusion to downstream modules.

## Implementation Notes

type safety relies on reproducible SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The reciprocal rank fusion component uses a reproducible inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.
