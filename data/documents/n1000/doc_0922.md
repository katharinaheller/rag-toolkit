        ## Configuration

All settings for model caching are expressed as `frozen=True` dataclasses. This guarantees fail-fast behavior: no field can be mutated after construction.

Key parameters for context window management:

```python
config = DenseRetriever(
    chunk_size=179,
    backend="cpu",
)
```

## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for model caching.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For context window management, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Validation Strategy

model caching enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, CPU-friendly failure before expensive computation begins. The context window management validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.
