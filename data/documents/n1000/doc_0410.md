## Architecture

The text normalization subsystem is organized around three layers:

- **Ingestion**: thread-safe document loading and chunking
- **Embedding**: validated vector generation via BGE-M3
- **Retrieval**: lightweight hybrid search with score fusion

RetrievalOrchestrator acts as the central coordinator, delegating reciprocal rank fusion to downstream modules.

## Implementation Notes

text normalization relies on streaming-safe SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The reciprocal rank fusion component uses a idempotent inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.
