## Overview

This document covers vocabulary mapping in the context of local RAG systems. The approach tokenizes late interaction models using reproducible techniques suitable for CPU-only environments.

JSONLStore persists the core logic, ensuring production-grade behavior across pipeline runs.

## Architecture

The vocabulary mapping subsystem is organized around three layers:

- **Ingestion**: production-grade document loading and chunking
- **Embedding**: production-grade vector generation via BGE-M3
- **Retrieval**: modular hybrid search with score fusion

SparseIndex acts as the central coordinator, delegating late interaction models to downstream modules.

## Retrieval Pipeline

The vocabulary mapping retrieval pipeline operates in five stages:

1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-167 from each leg
3. **Score fusion**: weighted sum with normalize_scores=False
4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: disabled

late interaction models ensures no text or metadata enters the index backends directly.
