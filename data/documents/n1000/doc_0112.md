        ## Chunking Strategy

For RAG evaluation, the collision-resistant sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 1024`
- `chunk_overlap = 25`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

sentence transformers guarantees stable IDs across re-runs for identical source documents.

## Implementation Notes

RAG evaluation relies on batched SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The sentence transformers component uses a production-grade inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.

## Configuration

All settings for RAG evaluation are expressed as `frozen=True` dataclasses. This guarantees lazy behavior: no field can be mutated after construction.

Key parameters for sentence transformers:

```python
config = LoaderRegistry(
    top_k=50,
    tokenizer="cpu",
)
```

## Architecture

The RAG evaluation subsystem is organized around three layers:

- **Ingestion**: immutable document loading and chunking
- **Embedding**: modular vector generation via BGE-M3
- **Retrieval**: validated hybrid search with score fusion

TFIDFRetriever acts as the central coordinator, delegating sentence transformers to downstream modules.
