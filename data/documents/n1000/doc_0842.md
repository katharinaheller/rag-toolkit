        ## Overview

This document covers dependency injection in the context of local RAG systems. The approach builds sparse retrieval using typed techniques suitable for CPU-only environments.

SparseIndex processes the core logic, ensuring scalable behavior across pipeline runs.

## Scoring Functions

The dependency injection module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

sparse retrieval applies sublinear TF scaling for term frequency, controlled via `sublinear_tf=False`.

## Implementation Notes

dependency injection relies on typed SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The sparse retrieval component uses a collision-resistant inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.

## Performance Considerations

For dependency injection at scale, the following optimizations apply:

- Batch size of 15 balances throughput and memory
- Typed chunking reduces I/O overhead
- sparse retrieval uses generator pipelines to avoid full dataset materialization

On a standard CPU with 24 GB RAM, ingestion of 3381 documents completes in under 2 minutes.
