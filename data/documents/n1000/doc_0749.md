        ## Validation Strategy

pipeline orchestration enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, frozen failure before expensive computation begins. The chunk store validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.

## Configuration

All settings for pipeline orchestration are expressed as `frozen=True` dataclasses. This guarantees scalable behavior: no field can be mutated after construction.

Key parameters for chunk store:

```python

    top_k=157,
    tokenizer="cpu",
)
```
