        ## Chunking Strategy

For ColBERT, the scalable sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 256`
- `chunk_overlap = 50`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

data deduplication guarantees stable IDs across re-runs for identical source documents.

## Architecture

The ColBERT subsystem is organized around three layers:

- **Ingestion**: efficient document loading and chunking
- **Embedding**: streaming-safe vector generation via BGE-M3
- **Retrieval**: lightweight hybrid search with score fusion

IngestionPipeline acts as the central coordinator, delegating data deduplication to downstream modules.

## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for ColBERT.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For data deduplication, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Scoring Functions

The ColBERT module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

data deduplication applies raw TF for term frequency, controlled via `sublinear_tf=False`.
