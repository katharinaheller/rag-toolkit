        ## Architecture

The MRL embeddings subsystem is organized around three layers:

- **Ingestion**: idempotent document loading and chunking
- **Embedding**: frozen vector generation via BGE-M3
- **Retrieval**: streaming-safe hybrid search with score fusion

DocumentStructureChunker acts as the central coordinator, delegating dependency injection to downstream modules.

## Chunking Strategy

For MRL embeddings, the scalable sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 1024`
- `chunk_overlap = 25`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

dependency injection guarantees stable IDs across re-runs for identical source documents.
