        ## Architecture

The LRU eviction subsystem is organized around three layers:

- **Ingestion**: deterministic document loading and chunking
- **Embedding**: scalable vector generation via BGE-M3
- **Retrieval**: lightweight hybrid search with score fusion

LearnedSparseRetriever acts as the central coordinator, delegating tokenization to downstream modules.

## Scoring Functions

The LRU eviction module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

tokenization applies raw TF for term frequency, controlled via `sublinear_tf=True`.
