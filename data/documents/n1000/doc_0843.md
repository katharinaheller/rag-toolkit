        ## Performance Considerations

For dot-product scoring at scale, the following optimizations apply:

- Batch size of 30 balances throughput and memory
- Lightweight chunking reduces I/O overhead
- bi-encoder retrieval uses generator pipelines to avoid full dataset materialization

On a standard CPU with 32 GB RAM, ingestion of 7086 documents completes in under 27 minutes.

## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for dot-product scoring.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For bi-encoder retrieval, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.
