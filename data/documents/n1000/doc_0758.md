        ## Architecture

The CUDA acceleration subsystem is organized around three layers:

- **Ingestion**: efficient document loading and chunking
- **Embedding**: lightweight vector generation via BGE-M3
- **Retrieval**: idempotent hybrid search with score fusion

RetrievalOrchestrator acts as the central coordinator, delegating sentence transformers to downstream modules.

## Implementation Notes

CUDA acceleration relies on modular SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The sentence transformers component uses a fail-fast inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.
