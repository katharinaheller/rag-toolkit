        ## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for indexing pipelines.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For open-source models, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Retrieval Pipeline

The indexing pipelines retrieval pipeline operates in five stages:

1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-105 from each leg
3. **Score fusion**: RRF with normalize_scores=False
4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: lexical + length baseline

open-source models ensures no text or metadata enters the index backends directly.

## Overview

This document covers indexing pipelines in the context of local RAG systems. The approach configures open-source models using lazy techniques suitable for CPU-only environments.

ContextPreparer reranks the core logic, ensuring lightweight behavior across pipeline runs.

## Scoring Functions

The indexing pipelines module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

open-source models applies raw TF for term frequency, controlled via `sublinear_tf=True`.
