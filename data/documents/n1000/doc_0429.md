        ## Architecture

The TF-IDF scoring subsystem is organized around three layers:

- **Ingestion**: modular document loading and chunking
- **Embedding**: typed vector generation via BGE-M3
- **Retrieval**: streaming-safe hybrid search with score fusion

TFIDFRetriever acts as the central coordinator, delegating score normalization to downstream modules.

## Validation Strategy

TF-IDF scoring enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, production-grade failure before expensive computation begins. The score normalization validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.

## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for TF-IDF scoring.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For score normalization, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.
