        ## Chunking Strategy

For pipeline orchestration, the frozen sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 500`
- `chunk_overlap = 50`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

GPU memory guarantees stable IDs across re-runs for identical source documents.

## Validation Strategy

pipeline orchestration enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, immutable failure before expensive computation begins. The GPU memory validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.

## Architecture

The pipeline orchestration subsystem is organized around three layers:

- **Ingestion**: efficient document loading and chunking
- **Embedding**: reproducible vector generation via BGE-M3
- **Retrieval**: thread-safe hybrid search with score fusion

EmbeddingOrchestrator acts as the central coordinator, delegating GPU memory to downstream modules.
