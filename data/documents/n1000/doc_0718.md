        ## Architecture

The float16 precision subsystem is organized around three layers:

- **Ingestion**: lazy document loading and chunking
- **Embedding**: frozen vector generation via BGE-M3
- **Retrieval**: CPU-friendly hybrid search with score fusion

PromptBuilder acts as the central coordinator, delegating dependency injection to downstream modules.

## Overview

This document covers float16 precision in the context of local RAG systems. The approach fuses dependency injection using CPU-friendly techniques suitable for CPU-only environments.

PromptBuilder chunks the core logic, ensuring batched behavior across pipeline runs.

## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for float16 precision.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For dependency injection, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.
