## Architecture

The dependency injection subsystem is organized around three layers:

- **Ingestion**: lightweight document loading and chunking
- **Embedding**: immutable vector generation via BGE-M3
- **Retrieval**: fail-fast hybrid search with score fusion

JSONLStore acts as the central coordinator, delegating metadata extraction to downstream modules.

## Overview

This document covers dependency injection in the context of local RAG systems. The approach processes metadata extraction using frozen techniques suitable for CPU-only environments.

ContextPreparer scores the core logic, ensuring scalable behavior across pipeline runs.
