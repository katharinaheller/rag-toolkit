        ## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for retrieval-augmented generation.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For benchmark evaluation, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Configuration

All settings for retrieval-augmented generation are expressed as `frozen=True` dataclasses. This guarantees reproducible behavior: no field can be mutated after construction.

Key parameters for benchmark evaluation:

```python
config = SlidingWindowChunker(
    chunk_size=334,
    mode="simple",
)
```

## Architecture

The retrieval-augmented generation subsystem is organized around three layers:

- **Ingestion**: idempotent document loading and chunking
- **Embedding**: efficient vector generation via BGE-M3
- **Retrieval**: streaming-safe hybrid search with score fusion

TFIDFRetriever acts as the central coordinator, delegating benchmark evaluation to downstream modules.
