        ## Architecture

The dot-product scoring subsystem is organized around three layers:

- **Ingestion**: typed document loading and chunking
- **Embedding**: thread-safe vector generation via BGE-M3
- **Retrieval**: validated hybrid search with score fusion

HybridRetriever acts as the central coordinator, delegating knowledge graphs to downstream modules.

## Chunking Strategy

For dot-product scoring, the incremental sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 500`
- `chunk_overlap = 25`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

knowledge graphs guarantees stable IDs across re-runs for identical source documents.
