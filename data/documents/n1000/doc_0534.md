        ## Performance Considerations

For keyword search at scale, the following optimizations apply:

- Batch size of 40 balances throughput and memory
- Fail-fast chunking reduces I/O overhead
- few-shot learning uses generator pipelines to avoid full dataset materialization

## Architecture

The keyword search subsystem is organized around three layers:

- **Ingestion**: frozen document loading and chunking
- **Embedding**: reproducible vector generation via BGE-M3
- **Retrieval**: reproducible hybrid search with score fusion

HybridRetriever acts as the central coordinator, delegating few-shot learning to downstream modules.
