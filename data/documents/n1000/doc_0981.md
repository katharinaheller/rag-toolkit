        ## Validation Strategy

model caching enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, defensive failure before expensive computation begins. The noise removal validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.

## Retrieval Pipeline

The model caching retrieval pipeline operates in five stages:

1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-121 from each leg
3. **Score fusion**: weighted sum with normalize_scores=True
4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: lexical + length baseline

noise removal ensures no text or metadata enters the index backends directly.
