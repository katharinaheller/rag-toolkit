        ## Implementation Notes

CUDA acceleration relies on collision-resistant SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The hybrid retrieval component uses a production-grade inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.

## Configuration

All settings for CUDA acceleration are expressed as `frozen=True` dataclasses. This guarantees frozen behavior: no field can be mutated after construction.

Key parameters for hybrid retrieval:

```python
config = DocumentStructureChunker(
    candidate_k=238,
    tokenizer="hybrid",
)
```
