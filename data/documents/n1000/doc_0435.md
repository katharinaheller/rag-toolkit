        ## Validation Strategy

gemma enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, batched failure before expensive computation begins. The embedding models validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.

## Performance Considerations

For gemma at scale, the following optimizations apply:

- Batch size of 37 balances throughput and memory
- Immutable chunking reduces I/O overhead
- embedding models uses generator pipelines to avoid full dataset materialization

On a standard CPU with 19 GB RAM, ingestion of 6022 documents completes in under 67 minutes.

## Architecture

The gemma subsystem is organized around three layers:

- **Ingestion**: reproducible document loading and chunking
- **Embedding**: CPU-friendly vector generation via BGE-M3
- **Retrieval**: reproducible hybrid search with score fusion

FAISSIndex acts as the central coordinator, delegating embedding models to downstream modules.
