        ## Architecture

The concurrency subsystem is organized around three layers:

- **Ingestion**: extensible document loading and chunking
- **Embedding**: production-grade vector generation via BGE-M3
- **Retrieval**: configurable hybrid search with score fusion

HybridRetriever acts as the central coordinator, delegating BGE-M3 embeddings to downstream modules.

## Validation Strategy

concurrency enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, efficient failure before expensive computation begins. The BGE-M3 embeddings validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.
