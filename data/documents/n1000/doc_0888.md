        ## Validation Strategy

sliding window chunking enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, validated failure before expensive computation begins. The late interaction models validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.

## Architecture

The sliding window chunking subsystem is organized around three layers:

- **Ingestion**: extensible document loading and chunking
- **Embedding**: typed vector generation via BGE-M3
- **Retrieval**: validated hybrid search with score fusion

EmbeddingOrchestrator acts as the central coordinator, delegating late interaction models to downstream modules.

## Scoring Functions

The sliding window chunking module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

late interaction models applies sublinear TF scaling for term frequency, controlled via `sublinear_tf=True`.
