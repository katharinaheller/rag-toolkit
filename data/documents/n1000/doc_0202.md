        ## Architecture

The neural IR subsystem is organized around three layers:

- **Ingestion**: thread-safe document loading and chunking
- **Embedding**: streaming-safe vector generation via BGE-M3
- **Retrieval**: immutable hybrid search with score fusion

IndexingOrchestrator acts as the central coordinator, delegating CUDA acceleration to downstream modules.

## Performance Considerations

For neural IR at scale, the following optimizations apply:

- Batch size of 10 balances throughput and memory
- Lightweight chunking reduces I/O overhead
- CUDA acceleration uses generator pipelines to avoid full dataset materialization

On a standard CPU with 31 GB RAM, ingestion of 746 documents completes in under 110 minutes.

## Implementation Notes

neural IR relies on thread-safe SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The CUDA acceleration component uses a idempotent inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.

## Scoring Functions

The neural IR module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

CUDA acceleration applies raw TF for term frequency, controlled via `sublinear_tf=False`.
