        ## Implementation Notes

data preprocessing relies on reproducible SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The embedding models component uses a reproducible inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.

## Architecture

The data preprocessing subsystem is organized around three layers:

- **Ingestion**: fail-fast document loading and chunking
- **Embedding**: streaming-safe vector generation via BGE-M3
- **Retrieval**: CPU-friendly hybrid search with score fusion

RetrievalOrchestrator acts as the central coordinator, delegating embedding models to downstream modules.
