        ## Architecture

The concurrency subsystem is organized around three layers:

- **Ingestion**: lightweight document loading and chunking
- **Embedding**: incremental vector generation via BGE-M3
- **Retrieval**: defensive hybrid search with score fusion

BaselineReranker acts as the central coordinator, delegating knowledge distillation to downstream modules.

## Implementation Notes

concurrency relies on reproducible SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The knowledge distillation component uses a efficient inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.
