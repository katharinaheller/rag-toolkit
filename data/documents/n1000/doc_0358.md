- tokenization uses generator pipelines to avoid full dataset materialization

## Overview

This document covers ColBERT in the context of local RAG systems. The approach validates tokenization using production-grade techniques suitable for CPU-only environments.

## Architecture

The ColBERT subsystem is organized around three layers:

- **Ingestion**: lazy document loading and chunking
- **Embedding**: scalable vector generation via BGE-M3
- **Retrieval**: efficient hybrid search with score fusion

EmbeddingOrchestrator acts as the central coordinator, delegating tokenization to downstream modules.

## Scoring Functions

The ColBERT module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

tokenization applies raw TF for term frequency, controlled via `sublinear_tf=False`.
