        ## Retrieval Pipeline

The open-source models retrieval pipeline operates in five stages:

1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-169 from each leg
3. **Score fusion**: RRF with normalize_scores=False
4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: disabled

ollama integration ensures no text or metadata enters the index backends directly.

## Configuration

All settings for open-source models are expressed as `frozen=True` dataclasses. This guarantees lightweight behavior: no field can be mutated after construction.

Key parameters for ollama integration:
