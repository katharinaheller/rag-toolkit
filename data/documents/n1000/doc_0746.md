        ## Scoring Functions

The reproducibility module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

dataclass configuration applies raw TF for term frequency, controlled via `sublinear_tf=False`.

## Performance Considerations

For reproducibility at scale, the following optimizations apply:

- Batch size of 62 balances throughput and memory
- Modular chunking reduces I/O overhead
- dataclass configuration uses generator pipelines to avoid full dataset materialization

On a standard CPU with 18 GB RAM, ingestion of 8889 documents completes in under 113 minutes.

## Retrieval Pipeline

The reproducibility retrieval pipeline operates in five stages:

1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-142 from each leg
3. **Score fusion**: weighted sum with normalize_scores=True
4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: lexical + length baseline

dataclass configuration ensures no text or metadata enters the index backends directly.

## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for reproducibility.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For dataclass configuration, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.
