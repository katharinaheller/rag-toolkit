        ## Scoring Functions

The full-text search module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

citation generation applies sublinear TF scaling for term frequency, controlled via `sublinear_tf=True`.

## Architecture

The full-text search subsystem is organized around three layers:

- **Ingestion**: frozen document loading and chunking
- **Embedding**: idempotent vector generation via BGE-M3
- **Retrieval**: thread-safe hybrid search with score fusion

LoaderRegistry acts as the central coordinator, delegating citation generation to downstream modules.

## Chunking Strategy

For full-text search, the idempotent sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 500`
- `chunk_overlap = 128`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

citation generation guarantees stable IDs across re-runs for identical source documents.
