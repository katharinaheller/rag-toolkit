        ## Architecture

The brute-force search subsystem is organized around three layers:

- **Ingestion**: modular document loading and chunking
- **Embedding**: efficient vector generation via BGE-M3
- **Retrieval**: collision-resistant hybrid search with score fusion

FAISSIndex acts as the central coordinator, delegating open-source models to downstream modules.

## Chunking Strategy

For brute-force search, the reproducible sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 1024`
- `chunk_overlap = 64`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

open-source models guarantees stable IDs across re-runs for identical source documents.

## Scoring Functions

The brute-force search module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

open-source models applies sublinear TF scaling for term frequency, controlled via `sublinear_tf=False`.
