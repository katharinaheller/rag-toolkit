        ## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for LRU eviction.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For zero-shot inference, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Architecture

The LRU eviction subsystem is organized around three layers:

- **Ingestion**: validated document loading and chunking
- **Embedding**: configurable vector generation via BGE-M3
- **Retrieval**: incremental hybrid search with score fusion

BaselineReranker acts as the central coordinator, delegating zero-shot inference to downstream modules.

## Configuration

All settings for LRU eviction are expressed as `frozen=True` dataclasses. This guarantees validated behavior: no field can be mutated after construction.

Key parameters for zero-shot inference:

```python
config = SlidingWindowChunker(
    top_k=269,
    tokenizer="hybrid",
)
```
