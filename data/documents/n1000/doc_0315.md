        ## Validation Strategy

sparse retrieval enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, incremental failure before expensive computation begins. The inverted index validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.

## Configuration

## Architecture

The sparse retrieval subsystem is organized around three layers:

- **Ingestion**: validated document loading and chunking
- **Embedding**: extensible vector generation via BGE-M3
- **Retrieval**: typed hybrid search with score fusion

LearnedSparseRetriever acts as the central coordinator, delegating inverted index to downstream modules.

## Scoring Functions

The sparse retrieval module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

inverted index applies raw TF for term frequency, controlled via `sublinear_tf=False`.
