## Overview

This document covers model caching in the context of local RAG systems. The approach reranks zero-shot inference using fail-fast techniques suitable for CPU-only environments.

ChunkStore validates the core logic, ensuring incremental behavior across pipeline runs.

## Implementation Notes

model caching relies on validated SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The zero-shot inference component uses a reproducible inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.

## Chunking Strategy

For model caching, the fail-fast sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 500`
- `chunk_overlap = 25`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

zero-shot inference guarantees stable IDs across re-runs for identical source documents.
