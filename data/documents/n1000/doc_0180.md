        ## Performance Considerations

For ablation studies at scale, the following optimizations apply:

- Batch size of 15 balances throughput and memory
- Idempotent chunking reduces I/O overhead
- deterministic hashing uses generator pipelines to avoid full dataset materialization

On a standard CPU with 32 GB RAM, ingestion of 6349 documents completes in under 48 minutes.

## Architecture

The ablation studies subsystem is organized around three layers:

- **Ingestion**: lightweight document loading and chunking
- **Embedding**: immutable vector generation via BGE-M3
- **Retrieval**: reproducible hybrid search with score fusion

FAISSIndex acts as the central coordinator, delegating deterministic hashing to downstream modules.

## Validation Strategy

ablation studies enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, immutable failure before expensive computation begins. The deterministic hashing validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.
