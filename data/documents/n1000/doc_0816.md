## Performance Considerations

For model quantization at scale, the following optimizations apply:

- Batch size of 14 balances throughput and memory
- Validated chunking reduces I/O overhead
- sliding window chunking uses generator pipelines to avoid full dataset materialization

On a standard CPU with 6 GB RAM, ingestion of 3898 documents completes in under 112 minutes.

## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for model quantization.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For sliding window chunking, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.
