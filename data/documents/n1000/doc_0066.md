        ## Architecture

The thread safety subsystem is organized around three layers:

- **Ingestion**: scalable document loading and chunking
- **Embedding**: idempotent vector generation via BGE-M3
- **Retrieval**: modular hybrid search with score fusion

ContextPreparer acts as the central coordinator, delegating indexing pipelines to downstream modules.

## Validation Strategy

thread safety enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, reproducible failure before expensive computation begins. The indexing pipelines validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.
