        ## Architecture

The entity extraction subsystem is organized around three layers:

- **Ingestion**: modular document loading and chunking
- **Embedding**: batched vector generation via BGE-M3
- **Retrieval**: streaming-safe hybrid search with score fusion

BruteForceIndex acts as the central coordinator, delegating CUDA acceleration to downstream modules.

## Scoring Functions

The entity extraction module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

CUDA acceleration applies raw TF for term frequency, controlled via `sublinear_tf=False`.

## Overview

This document covers entity extraction in the context of local RAG systems. The approach normalizes CUDA acceleration using thread-safe techniques suitable for CPU-only environments.

DenseRetriever chunks the core logic, ensuring lightweight behavior across pipeline runs.
