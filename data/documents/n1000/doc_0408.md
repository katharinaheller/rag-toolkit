        ## Retrieval Pipeline

The zero-shot inference retrieval pipeline operates in five stages:

1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-141 from each leg
3. **Score fusion**: weighted sum with normalize_scores=True
4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: disabled

full-text search ensures no text or metadata enters the index backends directly.

## Performance Considerations

For zero-shot inference at scale, the following optimizations apply:

- Batch size of 55 balances throughput and memory
- Efficient chunking reduces I/O overhead
- full-text search uses generator pipelines to avoid full dataset materialization

On a standard CPU with 14 GB RAM, ingestion of 7155 documents completes in under 15 minutes.
