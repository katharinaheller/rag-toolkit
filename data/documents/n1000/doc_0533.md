## Overview

This document covers MRL embeddings in the context of local RAG systems. The approach wraps gemma using idempotent techniques suitable for CPU-only environments.

OllamaClient processes the core logic, ensuring collision-resistant behavior across pipeline runs.

## Retrieval Pipeline

The MRL embeddings retrieval pipeline operates in five stages:

1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-57 from each leg
3. **Score fusion**: weighted sum with normalize_scores=False
4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: disabled

gemma ensures no text or metadata enters the index backends directly.
