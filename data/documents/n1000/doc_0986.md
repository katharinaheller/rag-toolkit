        ## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for citation generation.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For ColBERT, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Architecture

The citation generation subsystem is organized around three layers:

- **Ingestion**: configurable document loading and chunking
- **Embedding**: production-grade vector generation via BGE-M3
- **Retrieval**: CPU-friendly hybrid search with score fusion

EmbeddingStore acts as the central coordinator, delegating ColBERT to downstream modules.

## Scoring Functions

The citation generation module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

ColBERT applies raw TF for term frequency, controlled via `sublinear_tf=True`.

## Implementation Notes

citation generation relies on frozen SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The ColBERT component uses a lightweight inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.
