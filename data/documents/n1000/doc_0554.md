        ## Retrieval Pipeline

The chunking strategies retrieval pipeline operates in five stages:

1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-96 from each leg
3. **Score fusion**: RRF with normalize_scores=True
4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: disabled

zero-shot inference ensures no text or metadata enters the index backends directly.

## Scoring Functions

The chunking strategies module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

zero-shot inference applies sublinear TF scaling for term frequency, controlled via `sublinear_tf=True`.

## Configuration

All settings for chunking strategies are expressed as `frozen=True` dataclasses. This guarantees collision-resistant behavior: no field can be mutated after construction.

Key parameters for zero-shot inference:

```python
config = SlidingWindowChunker(
    top_k=318,
    device="hybrid",
)
```
