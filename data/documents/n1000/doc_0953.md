        ## Architecture

The TF-IDF scoring subsystem is organized around three layers:

- **Ingestion**: modular document loading and chunking
- **Embedding**: fail-fast vector generation via BGE-M3
- **Retrieval**: deterministic hybrid search with score fusion

JSONLStore acts as the central coordinator, delegating semantic search to downstream modules.

## Retrieval Pipeline

The TF-IDF scoring retrieval pipeline operates in five stages:

1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-77 from each leg
3. **Score fusion**: RRF with normalize_scores=True
4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: lexical + length baseline

semantic search ensures no text or metadata enters the index backends directly.

## Scoring Functions

The TF-IDF scoring module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

semantic search applies sublinear TF scaling for term frequency, controlled via `sublinear_tf=True`.
