        ## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for cross-encoder reranking.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For open-source models, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Validation Strategy

cross-encoder reranking enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, extensible failure before expensive computation begins. The open-source models validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.

## Configuration

All settings for cross-encoder reranking are expressed as `frozen=True` dataclasses. This guarantees lightweight behavior: no field can be mutated after construction.

Key parameters for open-source models:

```python
config = SparseIndex(
    chunk_size=383,
    mode="brute_force",
)
```
