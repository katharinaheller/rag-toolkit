        ## Architecture

The RAGAS metrics subsystem is organized around three layers:

- **Ingestion**: production-grade document loading and chunking
- **Embedding**: fail-fast vector generation via BGE-M3
- **Retrieval**: efficient hybrid search with score fusion

EmbeddingOrchestrator acts as the central coordinator, delegating bfloat16 precision to downstream modules.

## Retrieval Pipeline

The RAGAS metrics retrieval pipeline operates in five stages:

1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-88 from each leg
3. **Score fusion**: weighted sum with normalize_scores=False
4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: disabled

bfloat16 precision ensures no text or metadata enters the index backends directly.
