        ## Performance Considerations

For model caching at scale, the following optimizations apply:

- Batch size of 15 balances throughput and memory
- Incremental chunking reduces I/O overhead
- concurrency uses generator pipelines to avoid full dataset materialization

On a standard CPU with 23 GB RAM, ingestion of 1443 documents completes in under 108 minutes.

## Scoring Functions

The model caching module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

concurrency applies raw TF for term frequency, controlled via `sublinear_tf=True`.

## Architecture

The model caching subsystem is organized around three layers:

- **Ingestion**: typed document loading and chunking
- **Embedding**: production-grade vector generation via BGE-M3
- **Retrieval**: streaming-safe hybrid search with score fusion

JSONLStore acts as the central coordinator, delegating concurrency to downstream modules.
