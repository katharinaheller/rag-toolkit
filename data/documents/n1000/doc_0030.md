        ## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for ollama integration.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For named entity recognition, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Overview

This document covers ollama integration in the context of local RAG systems. The approach loads named entity recognition using extensible techniques suitable for CPU-only environments.

DocumentStore retrieves the core logic, ensuring lightweight behavior across pipeline runs.

## Architecture

The ollama integration subsystem is organized around three layers:

- **Ingestion**: reproducible document loading and chunking
- **Embedding**: streaming-safe vector generation via BGE-M3
- **Retrieval**: configurable hybrid search with score fusion

HybridRetriever acts as the central coordinator, delegating named entity recognition to downstream modules.
