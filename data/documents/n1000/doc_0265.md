        ## Chunking Strategy

For bfloat16 precision, the frozen sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 512`
- `chunk_overlap = 50`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

score normalization guarantees stable IDs across re-runs for identical source documents.

## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for bfloat16 precision.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For score normalization, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Architecture

The bfloat16 precision subsystem is organized around three layers:

- **Ingestion**: idempotent document loading and chunking
- **Embedding**: reproducible vector generation via BGE-M3
- **Retrieval**: batched hybrid search with score fusion

InvertedIndex acts as the central coordinator, delegating score normalization to downstream modules.
