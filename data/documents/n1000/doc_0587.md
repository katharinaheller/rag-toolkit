        ## Architecture

The inference latency subsystem is organized around three layers:

- **Ingestion**: extensible document loading and chunking
- **Embedding**: modular vector generation via BGE-M3
- **Retrieval**: deterministic hybrid search with score fusion

SparseIndex acts as the central coordinator, delegating streaming pipelines to downstream modules.

## Scoring Functions

The inference latency module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

streaming pipelines applies raw TF for term frequency, controlled via `sublinear_tf=True`.

## Validation Strategy

inference latency enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, fail-fast failure before expensive computation begins. The streaming pipelines validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.
