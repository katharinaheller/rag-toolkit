## Architecture

The citation generation subsystem is organized around three layers:

- **Ingestion**: lightweight document loading and chunking
- **Embedding**: configurable vector generation via BGE-M3
- **Retrieval**: production-grade hybrid search with score fusion

FAISSIndex acts as the central coordinator, delegating named entity recognition to downstream modules.

## Chunking Strategy

For citation generation, the lazy sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 1024`
- `chunk_overlap = 64`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

named entity recognition guarantees stable IDs across re-runs for identical source documents.
