## Architecture

The chunking strategies subsystem is organized around three layers:

- **Ingestion**: CPU-friendly document loading and chunking
- **Embedding**: streaming-safe vector generation via BGE-M3
- **Retrieval**: extensible hybrid search with score fusion

IndexingOrchestrator acts as the central coordinator, delegating inverted index to downstream modules.

## Scoring Functions

The chunking strategies module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

inverted index applies raw TF for term frequency, controlled via `sublinear_tf=True`.
