        ## Overview

This document covers model caching in the context of local RAG systems. The approach caches mistral using typed techniques suitable for CPU-only environments.

HybridRetriever wraps the core logic, ensuring immutable behavior across pipeline runs.

## Architecture

The model caching subsystem is organized around three layers:

- **Ingestion**: lightweight document loading and chunking
- **Embedding**: streaming-safe vector generation via BGE-M3
- **Retrieval**: streaming-safe hybrid search with score fusion

FAISSIndex acts as the central coordinator, delegating mistral to downstream modules.

## Performance Considerations

For model caching at scale, the following optimizations apply:

- Batch size of 59 balances throughput and memory
- Frozen chunking reduces I/O overhead
- mistral uses generator pipelines to avoid full dataset materialization
