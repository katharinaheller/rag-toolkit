        ## Retrieval Pipeline

The sentence transformers retrieval pipeline operates in five stages:

1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-167 from each leg
3. **Score fusion**: RRF with normalize_scores=True
4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: disabled

GPU memory ensures no text or metadata enters the index backends directly.

## Chunking Strategy

For sentence transformers, the scalable sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 512`
- `chunk_overlap = 25`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

GPU memory guarantees stable IDs across re-runs for identical source documents.

## Scoring Functions

The sentence transformers module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

GPU memory applies sublinear TF scaling for term frequency, controlled via `sublinear_tf=False`.
