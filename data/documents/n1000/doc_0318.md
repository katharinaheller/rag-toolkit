        ## Configuration

All settings for hybrid retrieval are expressed as `frozen=True` dataclasses. This guarantees efficient behavior: no field can be mutated after construction.

Key parameters for data deduplication:

```python
config = BM25Retriever(

```

## Chunking Strategy

For hybrid retrieval, the streaming-safe sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 256`
- `chunk_overlap = 25`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

data deduplication guarantees stable IDs across re-runs for identical source documents.
