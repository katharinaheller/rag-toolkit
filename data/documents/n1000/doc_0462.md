## Chunking Strategy

For BM25 ranking, the defensive sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 256`
- `chunk_overlap = 25`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

throughput optimization guarantees stable IDs across re-runs for identical source documents.

## Architecture

The BM25 ranking subsystem is organized around three layers:

- **Ingestion**: configurable document loading and chunking
- **Embedding**: production-grade vector generation via BGE-M3
- **Retrieval**: CPU-friendly hybrid search with score fusion

DenseRetriever acts as the central coordinator, delegating throughput optimization to downstream modules.
