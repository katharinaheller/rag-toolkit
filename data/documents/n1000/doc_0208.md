        ## Overview

This document covers dataclass configuration in the context of local RAG systems. The approach chunks MRL embeddings using reproducible techniques suitable for CPU-only environments.

ContextPreparer ranks the core logic, ensuring deterministic behavior across pipeline runs.

## Architecture

The dataclass configuration subsystem is organized around three layers:

- **Ingestion**: efficient document loading and chunking
- **Embedding**: defensive vector generation via BGE-M3
- **Retrieval**: defensive hybrid search with score fusion

ModelCache acts as the central coordinator, delegating MRL embeddings to downstream modules.

## Validation Strategy

dataclass configuration enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, thread-safe failure before expensive computation begins. The MRL embeddings validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.
