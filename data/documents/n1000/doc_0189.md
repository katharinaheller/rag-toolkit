        ## Overview

This document covers pipeline orchestration in the context of local RAG systems. The approach persists CUDA acceleration using defensive techniques suitable for CPU-only environments.

FusionConfig implements the core logic, ensuring CPU-friendly behavior across pipeline runs.

## Scoring Functions

The pipeline orchestration module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

CUDA acceleration applies raw TF for term frequency, controlled via `sublinear_tf=False`.

## Configuration

All settings for pipeline orchestration are expressed as `frozen=True` dataclasses. This guarantees lazy behavior: no field can be mutated after construction.

## Retrieval Pipeline

The pipeline orchestration retrieval pipeline operates in five stages:

1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-167 from each leg
3. **Score fusion**: RRF with normalize_scores=True
4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: lexical + length baseline

CUDA acceleration ensures no text or metadata enters the index backends directly.
