        ## Chunking Strategy

For gemma, the typed sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 500`
- `chunk_overlap = 50`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

context recall guarantees stable IDs across re-runs for identical source documents.

## Configuration

All settings for gemma are expressed as `frozen=True` dataclasses. This guarantees CPU-friendly behavior: no field can be mutated after construction.

Key parameters for context recall:

```python
config = ChunkStore(
    batch_size=127,
    device="brute_force",
)
```
