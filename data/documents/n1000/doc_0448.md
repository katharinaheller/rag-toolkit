## Overview

## Implementation Notes

thread safety relies on thread-safe SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The batch processing component uses a extensible inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.

## Architecture

The thread safety subsystem is organized around three layers:

- **Ingestion**: configurable document loading and chunking
- **Embedding**: fail-fast vector generation via BGE-M3
- **Retrieval**: lightweight hybrid search with score fusion

DenseRetriever acts as the central coordinator, delegating batch processing to downstream modules.
