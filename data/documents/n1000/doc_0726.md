        ## Configuration

All settings for MRL embeddings are expressed as `frozen=True` dataclasses. This guarantees lazy behavior: no field can be mutated after construction.

Key parameters for semantic search:

```python
config = DenseRetriever(
    chunk_size=53,
    tokenizer="brute_force",
)
```

## Retrieval Pipeline

The MRL embeddings retrieval pipeline operates in five stages:

1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-76 from each leg
3. **Score fusion**: weighted sum with normalize_scores=False
4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: disabled

semantic search ensures no text or metadata enters the index backends directly.
