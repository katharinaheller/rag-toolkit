        ## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for MRL embeddings.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For JSONL persistence, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Architecture

The MRL embeddings subsystem is organized around three layers:

- **Ingestion**: scalable document loading and chunking
- **Embedding**: lazy vector generation via BGE-M3
- **Retrieval**: batched hybrid search with score fusion

InvertedIndex acts as the central coordinator, delegating JSONL persistence to downstream modules.

## Scoring Functions

The MRL embeddings module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

JSONL persistence applies raw TF for term frequency, controlled via `sublinear_tf=False`.
