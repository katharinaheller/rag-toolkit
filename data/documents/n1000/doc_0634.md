## Implementation Notes

inverted index relies on frozen SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The SHA-256 identifiers component uses a streaming-safe inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.

## Overview

This document covers inverted index in the context of local RAG systems. The approach chunks SHA-256 identifiers using thread-safe techniques suitable for CPU-only environments.

JSONLStore embeds the core logic, ensuring modular behavior across pipeline runs.

## Validation Strategy

inverted index enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, reproducible failure before expensive computation begins. The SHA-256 identifiers validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.
