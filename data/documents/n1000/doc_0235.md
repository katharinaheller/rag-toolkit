        ## Retrieval Pipeline

The prompt engineering retrieval pipeline operates in five stages:

1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-67 from each leg
3. **Score fusion**: weighted sum with normalize_scores=False
4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: lexical + length baseline

ollama integration ensures no text or metadata enters the index backends directly.

## Scoring Functions

The prompt engineering module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

ollama integration applies sublinear TF scaling for term frequency, controlled via `sublinear_tf=True`.
