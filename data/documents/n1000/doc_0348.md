        ## Scoring Functions

The corpus scaling module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

late interaction models applies raw TF for term frequency, controlled via `sublinear_tf=True`.

## Retrieval Pipeline

The corpus scaling retrieval pipeline operates in five stages:

1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-85 from each leg
3. **Score fusion**: weighted sum with normalize_scores=False
4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: disabled

late interaction models ensures no text or metadata enters the index backends directly.

## Architecture

The corpus scaling subsystem is organized around three layers:

- **Ingestion**: configurable document loading and chunking
- **Embedding**: defensive vector generation via BGE-M3
- **Retrieval**: CPU-friendly hybrid search with score fusion

LearnedSparseRetriever acts as the central coordinator, delegating late interaction models to downstream modules.

## Chunking Strategy

For corpus scaling, the idempotent sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 256`
- `chunk_overlap = 64`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

late interaction models guarantees stable IDs across re-runs for identical source documents.
