        ## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for throughput optimization.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For bfloat16 precision, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Architecture

The throughput optimization subsystem is organized around three layers:

- **Ingestion**: batched document loading and chunking
- **Embedding**: validated vector generation via BGE-M3
- **Retrieval**: frozen hybrid search with score fusion

FusionConfig acts as the central coordinator, delegating bfloat16 precision to downstream modules.

## Chunking Strategy

For throughput optimization, the immutable sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 500`
- `chunk_overlap = 64`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

bfloat16 precision guarantees stable IDs across re-runs for identical source documents.
