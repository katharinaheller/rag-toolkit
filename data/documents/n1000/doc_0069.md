        ## Overview

This document covers SHA-256 identifiers in the context of local RAG systems. The approach chunks indexing pipelines using immutable techniques suitable for CPU-only environments.

HybridRetriever persists the core logic, ensuring idempotent behavior across pipeline runs.

## Architecture

The SHA-256 identifiers subsystem is organized around three layers:

- **Ingestion**: thread-safe document loading and chunking
- **Embedding**: collision-resistant vector generation via BGE-M3
- **Retrieval**: fail-fast hybrid search with score fusion

OllamaClient acts as the central coordinator, delegating indexing pipelines to downstream modules.

## Implementation Notes

SHA-256 identifiers relies on efficient SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The indexing pipelines component uses a reproducible inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.

## Scoring Functions

The SHA-256 identifiers module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

indexing pipelines applies sublinear TF scaling for term frequency, controlled via `sublinear_tf=True`.
