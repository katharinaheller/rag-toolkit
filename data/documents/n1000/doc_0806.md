        ## Implementation Notes

keyword search relies on frozen SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The deterministic hashing component uses a immutable inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.

## Performance Considerations

For keyword search at scale, the following optimizations apply:

- Batch size of 63 balances throughput and memory
- Typed chunking reduces I/O overhead
- deterministic hashing uses generator pipelines to avoid full dataset materialization

On a standard CPU with 22 GB RAM, ingestion of 4639 documents completes in under 66 minutes.
