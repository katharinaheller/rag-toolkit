        ## Chunking Strategy

For multimodal retrieval, the reproducible sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 1024`
- `chunk_overlap = 64`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

approximate nearest neighbors guarantees stable IDs across re-runs for identical source documents.

## Retrieval Pipeline

The multimodal retrieval retrieval pipeline operates in five stages:

1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-78 from each leg
3. **Score fusion**: RRF with normalize_scores=True
4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: disabled

approximate nearest neighbors ensures no text or metadata enters the index backends directly.
