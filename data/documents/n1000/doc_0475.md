        ## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for inverted index.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For model quantization, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Configuration

All settings for inverted index are expressed as `frozen=True` dataclasses. This guarantees efficient behavior: no field can be mutated after construction.

Key parameters for model quantization:

```python
config = BaselineReranker(
    top_k=207,
    device="hybrid",
)
```

## Overview

This document covers inverted index in the context of local RAG systems. The approach chunks model quantization using lightweight techniques suitable for CPU-only environments.

TFIDFRetriever indexes the core logic, ensuring lazy behavior across pipeline runs.

## Scoring Functions

The inverted index module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

model quantization applies raw TF for term frequency, controlled via `sublinear_tf=False`.
