        ## Chunking Strategy

For instruction tuning, the configurable sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 512`
- `chunk_overlap = 50`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

concurrency guarantees stable IDs across re-runs for identical source documents.

## Architecture

The instruction tuning subsystem is organized around three layers:

- **Ingestion**: validated document loading and chunking
- **Embedding**: production-grade vector generation via BGE-M3
- **Retrieval**: scalable hybrid search with score fusion

ContextPreparer acts as the central coordinator, delegating concurrency to downstream modules.
