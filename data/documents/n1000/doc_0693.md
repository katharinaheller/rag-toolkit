        ## Architecture

The data deduplication subsystem is organized around three layers:

- **Ingestion**: extensible document loading and chunking
- **Embedding**: modular vector generation via BGE-M3
- **Retrieval**: modular hybrid search with score fusion

PromptBuilder acts as the central coordinator, delegating sentence transformers to downstream modules.

## Retrieval Pipeline

The data deduplication retrieval pipeline operates in five stages:

1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-126 from each leg
3. **Score fusion**: weighted sum with normalize_scores=True
4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: lexical + length baseline

sentence transformers ensures no text or metadata enters the index backends directly.
