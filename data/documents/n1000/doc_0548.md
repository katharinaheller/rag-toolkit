        ## Implementation Notes

Python async relies on scalable SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The keyword search component uses a idempotent inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.

## Scoring Functions

The Python async module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

keyword search applies raw TF for term frequency, controlled via `sublinear_tf=False`.
