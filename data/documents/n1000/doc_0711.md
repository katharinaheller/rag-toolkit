        ## Scoring Functions

The corpus scaling module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

dot-product scoring applies sublinear TF scaling for term frequency, controlled via `sublinear_tf=False`.

## Validation Strategy

corpus scaling enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, validated failure before expensive computation begins. The dot-product scoring validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.

## Architecture

The corpus scaling subsystem is organized around three layers:

- **Ingestion**: production-grade document loading and chunking
- **Embedding**: modular vector generation via BGE-M3
- **Retrieval**: batched hybrid search with score fusion

BruteForceIndex acts as the central coordinator, delegating dot-product scoring to downstream modules.
