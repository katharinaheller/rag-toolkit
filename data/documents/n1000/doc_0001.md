## Implementation Notes

mistral relies on fail-fast SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The sentence transformers component uses a extensible inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.

## Chunking Strategy

For mistral, the reproducible sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 256`
- `chunk_overlap = 25`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

sentence transformers guarantees stable IDs across re-runs for identical source documents.

## Architecture

The mistral subsystem is organized around three layers:

- **Ingestion**: production-grade document loading and chunking
- **Embedding**: CPU-friendly vector generation via BGE-M3
- **Retrieval**: validated hybrid search with score fusion

OllamaClient acts as the central coordinator, delegating sentence transformers to downstream modules.
