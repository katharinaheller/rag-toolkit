        ## Scoring Functions

The FAISS indexing module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

chunking strategies applies sublinear TF scaling for term frequency, controlled via `sublinear_tf=False`.

## Validation Strategy

FAISS indexing enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, production-grade failure before expensive computation begins. The chunking strategies validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.

## Architecture

The FAISS indexing subsystem is organized around three layers:

- **Ingestion**: incremental document loading and chunking
- **Embedding**: idempotent vector generation via BGE-M3
- **Retrieval**: streaming-safe hybrid search with score fusion

InvertedIndex acts as the central coordinator, delegating chunking strategies to downstream modules.
