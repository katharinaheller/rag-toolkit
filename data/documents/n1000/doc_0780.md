        ## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for hallucination detection.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For data deduplication, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Architecture

The hallucination detection subsystem is organized around three layers:

- **Ingestion**: deterministic document loading and chunking
- **Embedding**: deterministic vector generation via BGE-M3
- **Retrieval**: defensive hybrid search with score fusion

IndexingOrchestrator acts as the central coordinator, delegating data deduplication to downstream modules.
