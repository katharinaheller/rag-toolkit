## Architecture

The knowledge distillation subsystem is organized around three layers:

- **Ingestion**: thread-safe document loading and chunking
- **Embedding**: thread-safe vector generation via BGE-M3
- **Retrieval**: immutable hybrid search with score fusion

JSONLStore acts as the central coordinator, delegating FAISS indexing to downstream modules.

## Validation Strategy

knowledge distillation enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, validated failure before expensive computation begins. The FAISS indexing validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.
