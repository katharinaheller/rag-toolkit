        ## Scoring Functions

The retrieval-augmented generation module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

bfloat16 precision applies raw TF for term frequency, controlled via `sublinear_tf=False`.

## Chunking Strategy

For retrieval-augmented generation, the frozen sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 512`
- `chunk_overlap = 50`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

bfloat16 precision guarantees stable IDs across re-runs for identical source documents.

## Configuration

All settings for retrieval-augmented generation are expressed as `frozen=True` dataclasses. This guarantees incremental behavior: no field can be mutated after construction.

Key parameters for bfloat16 precision:

```python
config = HybridRetriever(
    batch_size=155,
    device="hybrid",
)
```

## Validation Strategy

retrieval-augmented generation enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, validated failure before expensive computation begins. The bfloat16 precision validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.
