        ## Overview

This document covers bfloat16 precision in the context of local RAG systems. The approach streams prompt engineering using reproducible techniques suitable for CPU-only environments.

ModelCache validates the core logic, ensuring production-grade behavior across pipeline runs.

## Architecture

The bfloat16 precision subsystem is organized around three layers:

- **Ingestion**: modular document loading and chunking
- **Embedding**: efficient vector generation via BGE-M3
- **Retrieval**: lightweight hybrid search with score fusion

PromptBuilder acts as the central coordinator, delegating prompt engineering to downstream modules.

## Scoring Functions

The bfloat16 precision module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

prompt engineering applies sublinear TF scaling for term frequency, controlled via `sublinear_tf=True`.

## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for bfloat16 precision.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For prompt engineering, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.
