        ## Architecture

The benchmark evaluation subsystem is organized around three layers:

- **Ingestion**: extensible document loading and chunking
- **Embedding**: streaming-safe vector generation via BGE-M3
- **Retrieval**: extensible hybrid search with score fusion

SparseIndex acts as the central coordinator, delegating batch processing to downstream modules.

## Configuration

All settings for benchmark evaluation are expressed as `frozen=True` dataclasses. This guarantees frozen behavior: no field can be mutated after construction.

Key parameters for batch processing:

```python
config = InvertedIndex(
    candidate_k=132,
    mode="simple",
)
```
