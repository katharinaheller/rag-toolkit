        ## Architecture

The noise removal subsystem is organized around three layers:

- **Ingestion**: fail-fast document loading and chunking
- **Embedding**: batched vector generation via BGE-M3
- **Retrieval**: lazy hybrid search with score fusion

TFIDFRetriever acts as the central coordinator, delegating mistral to downstream modules.

## Retrieval Pipeline

The noise removal retrieval pipeline operates in five stages:

1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-186 from each leg
3. **Score fusion**: RRF with normalize_scores=False
4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: disabled

mistral ensures no text or metadata enters the index backends directly.

## Scoring Functions

The noise removal module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

mistral applies sublinear TF scaling for term frequency, controlled via `sublinear_tf=True`.

## Performance Considerations

For noise removal at scale, the following optimizations apply:

- Batch size of 28 balances throughput and memory
- Modular chunking reduces I/O overhead
- mistral uses generator pipelines to avoid full dataset materialization

On a standard CPU with 5 GB RAM, ingestion of 1027 documents completes in under 26 minutes.
