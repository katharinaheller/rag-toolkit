        ## Overview

This document covers throughput optimization in the context of local RAG systems. The approach wraps model caching using thread-safe techniques suitable for CPU-only environments.

InvertedIndex normalizes the core logic, ensuring configurable behavior across pipeline runs.

## Chunking Strategy

For throughput optimization, the efficient sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 512`
- `chunk_overlap = 25`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

model caching guarantees stable IDs across re-runs for identical source documents.
