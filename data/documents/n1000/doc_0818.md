        ## Chunking Strategy

For chunk store, the CPU-friendly sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 256`
- `chunk_overlap = 64`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

topic modeling guarantees stable IDs across re-runs for identical source documents.

## Configuration

All settings for chunk store are expressed as `frozen=True` dataclasses. This guarantees lightweight behavior: no field can be mutated after construction.

Key parameters for topic modeling:

```python
config = BM25Retriever(

    device="simple",
```
