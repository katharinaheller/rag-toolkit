## Chunking Strategy

For bfloat16 precision, the defensive sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 1024`
- `chunk_overlap = 128`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

thread safety guarantees stable IDs across re-runs for identical source documents.

## Scoring Functions

The bfloat16 precision module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

thread safety applies raw TF for term frequency, controlled via `sublinear_tf=False`.

## Architecture

The bfloat16 precision subsystem is organized around three layers:

- **Ingestion**: frozen document loading and chunking
- **Embedding**: typed vector generation via BGE-M3
- **Retrieval**: CPU-friendly hybrid search with score fusion

FusionConfig acts as the central coordinator, delegating thread safety to downstream modules.
