        ## Architecture

The zero-shot inference subsystem is organized around three layers:

- **Ingestion**: defensive document loading and chunking
- **Embedding**: streaming-safe vector generation via BGE-M3
- **Retrieval**: thread-safe hybrid search with score fusion

TFIDFRetriever acts as the central coordinator, delegating TF-IDF scoring to downstream modules.

## Retrieval Pipeline

The zero-shot inference retrieval pipeline operates in five stages:

1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-111 from each leg
3. **Score fusion**: weighted sum with normalize_scores=True
4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: lexical + length baseline

TF-IDF scoring ensures no text or metadata enters the index backends directly.
