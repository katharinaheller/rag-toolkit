        ## Scoring Functions

The embedding models module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

learned sparse representations applies sublinear TF scaling for term frequency, controlled via `sublinear_tf=True`.

## Retrieval Pipeline

The embedding models retrieval pipeline operates in five stages:

1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-193 from each leg
3. **Score fusion**: RRF with normalize_scores=False
4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: lexical + length baseline

learned sparse representations ensures no text or metadata enters the index backends directly.

## Performance Considerations

For embedding models at scale, the following optimizations apply:

- Batch size of 49 balances throughput and memory
- Frozen chunking reduces I/O overhead
- learned sparse representations uses generator pipelines to avoid full dataset materialization

On a standard CPU with 15 GB RAM, ingestion of 5207 documents completes in under 108 minutes.
