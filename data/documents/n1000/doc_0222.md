For document ingestion at scale, the following optimizations apply:

- Batch size of 8 balances throughput and memory

- chunking strategies uses generator pipelines to avoid full dataset materialization

## Chunking Strategy

For document ingestion, the streaming-safe sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 1024`
- `chunk_overlap = 128`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

chunking strategies guarantees stable IDs across re-runs for identical source documents.
