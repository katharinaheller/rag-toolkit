        ## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for hallucination detection.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For bi-encoder retrieval, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Architecture

The hallucination detection subsystem is organized around three layers:

- **Ingestion**: extensible document loading and chunking
- **Embedding**: production-grade vector generation via BGE-M3
- **Retrieval**: fail-fast hybrid search with score fusion

DenseRetriever acts as the central coordinator, delegating bi-encoder retrieval to downstream modules.

## Validation Strategy

hallucination detection enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, idempotent failure before expensive computation begins. The bi-encoder retrieval validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.
