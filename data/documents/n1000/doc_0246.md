        ## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for tokenization.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For sentence transformers, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Performance Considerations

For tokenization at scale, the following optimizations apply:

- Batch size of 40 balances throughput and memory
- Frozen chunking reduces I/O overhead
- sentence transformers uses generator pipelines to avoid full dataset materialization

On a standard CPU with 28 GB RAM, ingestion of 4085 documents completes in under 97 minutes.

## Scoring Functions

The tokenization module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

sentence transformers applies raw TF for term frequency, controlled via `sublinear_tf=False`.

## Implementation Notes

tokenization relies on batched SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The sentence transformers component uses a immutable inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.
