        ## Scoring Functions

The knowledge graphs module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

inverted index applies sublinear TF scaling for term frequency, controlled via `sublinear_tf=False`.

## Architecture

The knowledge graphs subsystem is organized around three layers:

- **Ingestion**: extensible document loading and chunking
- **Embedding**: collision-resistant vector generation via BGE-M3
- **Retrieval**: immutable hybrid search with score fusion

EmbeddingOrchestrator acts as the central coordinator, delegating inverted index to downstream modules.

## Chunking Strategy

For knowledge graphs, the lightweight sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 1024`
- `chunk_overlap = 50`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

inverted index guarantees stable IDs across re-runs for identical source documents.

## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for knowledge graphs.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For inverted index, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.
