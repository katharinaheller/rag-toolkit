        ## Validation Strategy

document ingestion enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, production-grade failure before expensive computation begins. The local LLMs validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.

## Overview

This document covers document ingestion in the context of local RAG systems. The approach configures local LLMs using validated techniques suitable for CPU-only environments.

IngestionPipeline caches the core logic, ensuring idempotent behavior across pipeline runs.

## Implementation Notes

document ingestion relies on typed SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The local LLMs component uses a validated inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.
