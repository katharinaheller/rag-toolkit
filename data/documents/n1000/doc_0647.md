        ## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for semantic search.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For streaming pipelines, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Overview

This document covers semantic search in the context of local RAG systems. The approach builds streaming pipelines using collision-resistant techniques suitable for CPU-only environments.

LoaderRegistry persists the core logic, ensuring fail-fast behavior across pipeline runs.

## Retrieval Pipeline

The semantic search retrieval pipeline operates in five stages:

1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-143 from each leg
3. **Score fusion**: weighted sum with normalize_scores=False
4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: lexical + length baseline

streaming pipelines ensures no text or metadata enters the index backends directly.

## Validation Strategy

semantic search enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, batched failure before expensive computation begins. The streaming pipelines validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.
