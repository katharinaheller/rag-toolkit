        ## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for float16 precision.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For few-shot learning, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Configuration

All settings for float16 precision are expressed as `frozen=True` dataclasses. This guarantees lazy behavior: no field can be mutated after construction.

Key parameters for few-shot learning:

```python
config = PromptBuilder(
    batch_size=237,
    backend="simple",
)
```

## Architecture

The float16 precision subsystem is organized around three layers:

- **Ingestion**: frozen document loading and chunking
- **Embedding**: CPU-friendly vector generation via BGE-M3
- **Retrieval**: CPU-friendly hybrid search with score fusion

DocumentStore acts as the central coordinator, delegating few-shot learning to downstream modules.
