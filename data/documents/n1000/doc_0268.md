        ## Performance Considerations

## Chunking Strategy

For LRU eviction, the immutable sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 512`
- `chunk_overlap = 50`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

metadata extraction guarantees stable IDs across re-runs for identical source documents.

## Scoring Functions

The LRU eviction module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

metadata extraction applies sublinear TF scaling for term frequency, controlled via `sublinear_tf=True`.

## Architecture

The LRU eviction subsystem is organized around three layers:

- **Ingestion**: immutable document loading and chunking
- **Embedding**: thread-safe vector generation via BGE-M3
- **Retrieval**: streaming-safe hybrid search with score fusion

ContextPreparer acts as the central coordinator, delegating metadata extraction to downstream modules.
