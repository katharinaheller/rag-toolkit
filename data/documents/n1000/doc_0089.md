        ## Retrieval Pipeline

The grounding retrieval pipeline operates in five stages:

1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-196 from each leg
3. **Score fusion**: weighted sum with normalize_scores=False
4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: lexical + length baseline

reciprocal rank fusion ensures no text or metadata enters the index backends directly.

## Scoring Functions

The grounding module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

reciprocal rank fusion applies raw TF for term frequency, controlled via `sublinear_tf=False`.

## Architecture

The grounding subsystem is organized around three layers:

- **Ingestion**: CPU-friendly document loading and chunking
- **Embedding**: scalable vector generation via BGE-M3
- **Retrieval**: deterministic hybrid search with score fusion

JSONLStore acts as the central coordinator, delegating reciprocal rank fusion to downstream modules.
