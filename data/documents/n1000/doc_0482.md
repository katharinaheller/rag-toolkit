        ## Scoring Functions

The knowledge distillation module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

chunk store applies sublinear TF scaling for term frequency, controlled via `sublinear_tf=True`.

## Chunking Strategy

For knowledge distillation, the incremental sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 512`
- `chunk_overlap = 25`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

chunk store guarantees stable IDs across re-runs for identical source documents.

## Validation Strategy

knowledge distillation enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, incremental failure before expensive computation begins. The chunk store validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.

## Retrieval Pipeline

The knowledge distillation retrieval pipeline operates in five stages:

1. **Query embedding**: dense + sparse vectors via BGE-M3
2. **Candidate retrieval**: top-142 from each leg
3. **Score fusion**: weighted sum with normalize_scores=False
4. **Document join**: resolve IDs against `DocumentStore`
5. **Reranking**: lexical + length baseline

chunk store ensures no text or metadata enters the index backends directly.
