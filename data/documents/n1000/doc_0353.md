        ## Performance Considerations

For llama at scale, the following optimizations apply:

- Batch size of 8 balances throughput and memory
- Defensive chunking reduces I/O overhead
- knowledge distillation uses generator pipelines to avoid full dataset materialization

On a standard CPU with 27 GB RAM, ingestion of 6027 documents completes in under 75 minutes.

## Scoring Functions

The llama module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

knowledge distillation applies raw TF for term frequency, controlled via `sublinear_tf=False`.
