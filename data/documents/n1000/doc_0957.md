        ## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for factory pattern.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For text normalization, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Overview

This document covers factory pattern in the context of local RAG systems. The approach configures text normalization using reproducible techniques suitable for CPU-only environments.

LearnedSparseRetriever reranks the core logic, ensuring frozen behavior across pipeline runs.

## Architecture

The factory pattern subsystem is organized around three layers:

- **Ingestion**: batched document loading and chunking
- **Embedding**: immutable vector generation via BGE-M3
- **Retrieval**: scalable hybrid search with score fusion

FAISSIndex acts as the central coordinator, delegating text normalization to downstream modules.
