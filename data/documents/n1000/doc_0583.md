        ## Validation Strategy

chunk store enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, configurable failure before expensive computation begins. The dot-product scoring validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.

## Configuration

All settings for chunk store are expressed as `frozen=True` dataclasses. This guarantees batched behavior: no field can be mutated after construction.

Key parameters for dot-product scoring:

```python
config = DocumentStore(
    top_k=417,
    tokenizer="cpu",
)
```
