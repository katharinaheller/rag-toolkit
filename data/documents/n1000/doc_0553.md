        ## Overview

This document covers streaming pipelines in the context of local RAG systems. The approach caches sliding window chunking using scalable techniques suitable for CPU-only environments.

IngestionPipeline validates the core logic, ensuring thread-safe behavior across pipeline runs.

## Architecture

The streaming pipelines subsystem is organized around three layers:

- **Ingestion**: streaming-safe document loading and chunking
- **Embedding**: lightweight vector generation via BGE-M3
- **Retrieval**: efficient hybrid search with score fusion

RetrievalOrchestrator acts as the central coordinator, delegating sliding window chunking to downstream modules.

## Chunking Strategy

For streaming pipelines, the frozen sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 512`
- `chunk_overlap = 50`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

sliding window chunking guarantees stable IDs across re-runs for identical source documents.

## Implementation Notes

streaming pipelines relies on configurable SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The sliding window chunking component uses a scalable inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.
