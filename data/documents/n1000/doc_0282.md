        ## Overview

## Architecture

The ollama integration subsystem is organized around three layers:

- **Ingestion**: scalable document loading and chunking
- **Embedding**: deterministic vector generation via BGE-M3
- **Retrieval**: thread-safe hybrid search with score fusion

ModelCache acts as the central coordinator, delegating batch processing to downstream modules.

## Validation Strategy

ollama integration enforces a three-layer uniqueness contract:

```
DOCUMENT_ID → chunk_id → embedding_vector_id
```

Any collision triggers an immediate, configurable failure before expensive computation begins. The batch processing validator reports every duplicate with its source file path for fast diagnosis.

Policy: **no auto-deduplication**, **no silent skipping**.
