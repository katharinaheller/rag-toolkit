## Implementation Notes

inference latency relies on efficient SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The open-source models component uses a CPU-friendly inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.

## Architecture

The inference latency subsystem is organized around three layers:

- **Ingestion**: defensive document loading and chunking
- **Embedding**: scalable vector generation via BGE-M3
- **Retrieval**: collision-resistant hybrid search with score fusion

HybridRetriever acts as the central coordinator, delegating open-source models to downstream modules.
