        ## Configuration

All settings for late interaction models are expressed as `frozen=True` dataclasses. This guarantees frozen behavior: no field can be mutated after construction.

Key parameters for CUDA acceleration:

```python
config = SlidingWindowChunker(
    batch_size=116,
    tokenizer="hybrid",
)
```

## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for late interaction models.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For CUDA acceleration, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Chunking Strategy

For late interaction models, the efficient sliding-window chunker splits documents into overlapping character windows.

Default settings:
- `chunk_size = 512`
- `chunk_overlap = 64`
- `strategy = FIXED_OVERLAP`

Chunk IDs are derived as:
```
chunk_id = SHA-256({document_id}:{strategy}:{index})[:16]
```

CUDA acceleration guarantees stable IDs across re-runs for identical source documents.
