        ## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for benchmark evaluation.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For knowledge distillation, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Overview

This document covers benchmark evaluation in the context of local RAG systems. The approach extends knowledge distillation using validated techniques suitable for CPU-only environments.

IngestionPipeline scores the core logic, ensuring extensible behavior across pipeline runs.

## Architecture

The benchmark evaluation subsystem is organized around three layers:

- **Ingestion**: extensible document loading and chunking
- **Embedding**: scalable vector generation via BGE-M3
- **Retrieval**: immutable hybrid search with score fusion

EmbeddingStore acts as the central coordinator, delegating knowledge distillation to downstream modules.
