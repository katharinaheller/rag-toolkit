        ## Performance Considerations

For MRL embeddings at scale, the following optimizations apply:

- Batch size of 46 balances throughput and memory
- Collision-resistant chunking reduces I/O overhead
- data preprocessing uses generator pipelines to avoid full dataset materialization

On a standard CPU with 14 GB RAM, ingestion of 3550 documents completes in under 59 minutes.

## Scoring Functions

The MRL embeddings module provides pure, stateless scoring functions:

**BM25** — Robertson-Sparck Jones smoothed IDF:
```
IDF(t) = log((N - df(t) + 0.5) / (df(t) + 0.5) + 1)
```

**TF-IDF** — smoothed variant:
```
IDF(t) = log((N + 1) / (df(t) + 1)) + 1
```

data preprocessing applies raw TF for term frequency, controlled via `sublinear_tf=True`.

## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for MRL embeddings.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For data preprocessing, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.

## Architecture

The MRL embeddings subsystem is organized around three layers:

- **Ingestion**: lazy document loading and chunking
- **Embedding**: modular vector generation via BGE-M3
- **Retrieval**: incremental hybrid search with score fusion

EmbeddingOrchestrator acts as the central coordinator, delegating data preprocessing to downstream modules.
