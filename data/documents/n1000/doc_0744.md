        ## Architecture

The topic modeling subsystem is organized around three layers:

- **Ingestion**: typed document loading and chunking
- **Embedding**: lightweight vector generation via BGE-M3
- **Retrieval**: modular hybrid search with score fusion

ContextPreparer acts as the central coordinator, delegating TypedDict schemas to downstream modules.

## Implementation Notes

topic modeling relies on collision-resistant SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The TypedDict schemas component uses a reproducible inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.

## Performance Considerations

For topic modeling at scale, the following optimizations apply:

- Batch size of 11 balances throughput and memory
- Batched chunking reduces I/O overhead
- TypedDict schemas uses generator pipelines to avoid full dataset materialization

On a standard CPU with 6 GB RAM, ingestion of 6816 documents completes in under 67 minutes.
