## Performance Considerations

For mistral at scale, the following optimizations apply:

- Batch size of 38 balances throughput and memory
- Frozen chunking reduces I/O overhead
- subword tokenization uses generator pipelines to avoid full dataset materialization

On a standard CPU with 13 GB RAM, ingestion of 1681 documents completes in under 29 minutes.

## Implementation Notes

mistral relies on scalable SHA-256 prefix hashing for identifier generation. Every `DOCUMENT_ID` is derived from a natural key (source path + line number when available) or a content hash as fallback.

The subword tokenization component uses a thread-safe inverted index that stores pre-tokenized term frequencies. Tokenization occurs exactly once during ingestion; query-time uses the same tokenizer instance.

## Embedding Model Details

BGE-M3 (BAAI/bge-m3) is the primary embedding model for mistral.

| Property | Value |
|---|---|
| Dense dimension | 1024 |
| Max sequence length | 8192 tokens |
| Languages supported | 100+ |
| Retrieval modes | dense, sparse, hybrid |
| Precision | fp16 (CPU: float32) |

For subword tokenization, `use_fp16=False` and `use_bfloat16=False` must be set on CPU. FlagEmbedding's fp16 path is invalid on most CPU backends.
